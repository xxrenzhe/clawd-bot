## Step-by-Step: Setting Up Your Business AI Gateway

Ready to build your own? Don't worry if you aren't a "hardcore" developer. If you can copy-paste a command into a terminal, you can do this. You'll need a small VPS (Virtual Private Server) or even an old computer under your desk running Linux.

### 1. The Quick Installation
First, we need to get the Moltbot binaries onto your system. Open your terminal and run the verified installation script:

```bash
curl -fsSL https://clawd.bot/install.sh | bash
```

This script detects your environment and installs the necessary components to get Openclaw running.

### 2. The Onboarding Process
Once installed, you need to configure your bot. This is where you tell Moltbot which AI models you want to use (like Claude 3.5) and which platforms you want to chat on.

Run:
```bash
clawdbot onboard
```
Follow the interactive prompts. You will be asked for your API keys. If you’re focused on high-quality business output, I highly recommend using a **Claude API key** from Anthropic—it currently offers the best balance of reasoning and cost.

### 3. Verification and Health Check
Before we go live, let's make sure everything is connected correctly. 

```bash
clawdbot health
```
If you see green checkmarks, you’re ready to roll. If something is red, double-check your API keys and internet connection.

### 4. Launching the Gateway
The gateway is the "brain" that stays awake and listens for incoming messages from your team. To start it on a specific port with detailed logging (great for troubleshooting), use:

```bash
clawdbot gateway --port 18789 --verbose
```

## Security: Protecting Your Business Data

When you host your own AI, **you** are the IT department. While Moltbot is designed to be secure, you must follow these best practices to prevent unauthorized access to your expensive API keys.

### Bind to Loopback
By default, you don't want the whole world to be able to ping your AI gateway. In your configuration file, ensure your `gateway.bind` is set to "loopback" (127.0.0.1). This means only services on your actual server (like a reverse proxy) can talk to it.

### Pairing Mode
Moltbot features a "Pairing" mode for DM (Direct Message) policies. I strongly recommend enabling this. When a new team member tries to message the bot on Telegram, the bot won't respond until you manually "pair" or approve that device.

### Use a Password
Never leave your gateway wide open. Set a strong password in your config:
```yaml
gateway:
  auth:
    password: "your-super-secure-password-here"
```

### Trusted Proxies
If you are using a tool like Nginx or Cloudflare Tunnel to access your bot from the web, make sure to configure `gateway.trustedProxies`. This ensures that Moltbot knows which incoming traffic is legitimate and which is a potential spoofing attempt.

How are actual SMEs using **Clawdbot** today? Here are three practical examples:

## Common Hurdles (The Honest Truth)

Self-hosting isn't always "set it and forget it." Here are a few things you might run into:
*   **Server Maintenance:** You need to keep your Linux VPS updated.
*   **API Limits:** If your team goes crazy with long prompts, you might hit your Anthropic or OpenAI rate limits. Monitor your usage dashboard!
*   **Configuration Syntax:** YAML files are picky. One wrong space and the bot won't start. (Pro tip: use a YAML validator if you're stuck).
