## Understanding the Moltbot Architecture

Before diving into the commands, it is essential to understand what makes Moltbot unique. Unlike standard bots that simply relay text, Moltbot functions as a "Universal AI Gateway." 

### The Core Components
1.  **The Gateway:** The heart of the system that manages connections between messaging platforms (Telegram, Discord, Slack, WhatsApp) and LLM providers.
2.  **Providers:** Moltbot supports industry leaders like Anthropic (Claude) and OpenAI (GPT-4), as well as local providers like Ollama for those who prefer 100% offline operations.
3.  **Skills & Plugins:** These are the "hands" of the AI. Skills allow the bot to read your filesystem, execute code, or fetch real-time data from the web.
4.  **Memory Persistence:** Unlike standard API calls that lose context, Moltbot uses file-based memory persistence to ensure your assistant remembers previous interactions and project details.

By combining these elements, you can move beyond "asking questions" and start "assigning tasks."

## Getting Started: Installation and Onboarding

Moltbot is designed for self-hosting. This ensures that your API keys and private conversations never sit on a third-party server. To begin, you will need a Linux-based environment (VPS or local machine).

### Quick Installation
Run the following verified command to download and install the Moltbot environment:

```bash
curl -fsSL https://clawd.bot/install.sh | bash
```

This script handles the dependencies and sets up the `clawdbot` binary. Once the installation is complete, you need to configure your identity and connect your first LLM provider.

### The Onboarding Process
Run the interactive onboarding tool:

```bash
clawdbot onboard
```

During this process, you will be prompted to:
*   Select your primary messaging platform (e.g., Telegram).
*   Input your LLM API keys (Claude is highly recommended for complex workflows).
*   Define your "Master" user ID to ensure only you can control the bot.

### Verifying System Health
Before deploying complex workflows, verify that everything is configured correctly:

```bash
clawdbot health
```

If all systems show a "Healthy" status, you are ready to launch the gateway.

## Launching the AI Gateway

The gateway is the persistent process that listens for incoming messages and routes them to the AI. For production-like workflows, you typically want to run this with high verbosity to monitor how the AI is interpreting your commands.

```bash
clawdbot gateway --port 18789 --verbose
```

This command opens the gateway on port 18789. The `--verbose` flag is crucial during the workflow development phase as it provides insight into the "Chain of Thought" the AI uses before executing a skill.

## Building Intelligent Workflows

The true power of Moltbot lies in **Intelligent Automation**. Instead of writing rigid "If-This-Then-That" rules, you describe the desired outcome to the AI, and it uses its available skills to fulfill the request.

### 1. File-Based Knowledge Management
One of the most common workflows involves managing a local repository of documents. Because Moltbot has filesystem access, you can use it to summarize local logs or draft documentation.

**Example Workflow:**
*   **User Command:** "Read the latest log file in `/var/logs/app/` and tell me if there were any critical errors in the last hour."
*   **AI Action:** The bot uses the `file_read` skill, parses the timestamps, identifies "CRITICAL" strings, and sends a summarized report back to your Telegram.

### 2. Multi-Platform Synchronization
Moltbot can bridge different messaging ecosystems. You can configure a workflow where the AI monitors a specific Slack channel and forwards relevant technical updates to a private Discord server, adding its own analysis in the process.

### 3. Proactive Automation with Cron
By integrating Moltbot with system-level cron jobs, you can create "Proactive AI."
*   **Workflow:** At 9:00 AM every morning, a cron job triggers a Moltbot script that fetches your calendar, checks the weather, and sends a "Daily Briefing" to your WhatsApp with suggestions on when to schedule deep-work sessions.

## Security Best Practices for Self-Hosted AI

When you give an AI access to your computer and messaging apps, security is paramount. Moltbot includes several built-in features to protect your instance.

### Restricting Network Exposure
By default, you should ensure the gateway is not accessible to the public internet unless behind a secure proxy.
*   **Loopback Binding:** Bind the gateway to the "loopback" interface (127.0.0.1) so only internal services can talk to it.
*   **Trusted Proxies:** If you use Nginx or Caddy, configure the `gateway.trustedProxies` setting in your configuration file to only accept traffic from your reverse proxy.

### Authentication and Pairing
Never leave your gateway open without a password.
1.  **Gateway Password:** Set `gateway.auth.password` in your config to prevent unauthorized API access.
2.  **Pairing Mode:** Use "pairing" mode for Direct Message (DM) policies. This requires you to manually approve any new device or user that attempts to interact with the bot.

### Sandboxing Tools
If you are using Moltbot in a group chat (like a team Slack), use the "Sandbox" feature for tools. This prevents the AI from executing sensitive file-system commands unless specifically authorized by an administrator.

## Extending Moltbot: Custom Skills

For intermediate users, the real fun begins with writing custom skills. A skill is essentially a JavaScript or Python function that the AI can choose to call.

If you need a workflow that interacts with a specific internal database, you can write a skill that:
1.  Connects to the database.
2.  Sanitizes the AI's query.
3.  Returns the results in a structured format.

This turns Moltbot into a custom Business Intelligence (BI) tool that you can talk to.
