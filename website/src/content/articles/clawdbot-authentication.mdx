# Moltbot Authentication: Secure Your Gateway

![Moltbot Authentication Setup](/images/articles/clawdbot-authentication.jpg)

## Authentication Methods

| Method | Best For | Security |
|--------|----------|----------|
| Password | Dashboard access | Medium |
| API Key | Programmatic access | High |
| OAuth | User management | High |
| Pairing | Messaging channels | High |

<HostingCTA context="setup" />

## Dashboard Password

### Set Password

```bash
moltbot config set gateway.auth.password "your-strong-password"
```

Or in config file:

```json
{
  "gateway": {
    "auth": {
      "password": "your-strong-password"
    }
  }
}
```

### Access Dashboard

Navigate to `http://localhost:18789` and enter password when prompted.

## API Key Authentication

### Generate API Key

```bash
moltbot auth create-key --name "my-app"
```

Output:
```
API Key: mb_live_abc123xyz...
Name: my-app
Created: 2026-01-28

Store this key securely. It won't be shown again.
```

### Use API Key

```bash
# In requests
curl -H "Authorization: Bearer mb_live_abc123xyz" \
  http://localhost:18789/chat

# In environment
export MOLTBOT_API_KEY="mb_live_abc123xyz"
```

### Manage Keys

```bash
# List keys
moltbot auth list-keys

# Revoke key
moltbot auth revoke-key mb_live_abc123xyz

# Rotate key
moltbot auth rotate-key mb_live_abc123xyz
```

## User Access Control

### Configure Allowed Users

```json
{
  "gateway": {
    "auth": {
      "allowedUsers": [
        "telegram:123456789",
        "discord:987654321"
      ]
    }
  }
}
```

### DM Policy

```json
{
  "channels": {
    "telegram": {
      "dmPolicy": "pairing"
    }
  }
}
```

DM policies:
- `open`: Anyone can message (not recommended)
- `pairing`: Require pairing code
- `whitelist`: Only allowed users

### Pairing Mode

```bash
# Generate pairing code
moltbot auth pair

# Output
Pairing code: ABC123
Expires in: 5 minutes

# User sends to bot
/pair ABC123
```

## Trusted Proxies

### Configure for Reverse Proxy

```json
{
  "gateway": {
    "trustedProxies": ["127.0.0.1", "10.0.0.0/8"],
    "auth": {
      "trustProxy": true
    }
  }
}
```

### Nginx Auth Header

```nginx
location / {
    proxy_pass http://127.0.0.1:18789;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Real-IP $remote_addr;
}
```

## OAuth Integration

### Google OAuth

```json
{
  "gateway": {
    "auth": {
      "oauth": {
        "provider": "google",
        "clientId": "your-client-id",
        "clientSecret": "your-client-secret",
        "allowedDomains": ["yourcompany.com"]
      }
    }
  }
}
```

### GitHub OAuth

```json
{
  "gateway": {
    "auth": {
      "oauth": {
        "provider": "github",
        "clientId": "your-client-id",
        "clientSecret": "your-client-secret",
        "allowedOrgs": ["your-org"]
      }
    }
  }
}
```

## Session Management

### Session Config

```json
{
  "gateway": {
    "auth": {
      "session": {
        "secret": "random-secret-key",
        "maxAge": 86400,
        "secure": true
      }
    }
  }
}
```

### Clear Sessions

```bash
# Clear all sessions
moltbot auth clear-sessions

# Clear specific user
moltbot auth clear-sessions --user "telegram:123456789"
```

## Two-Factor Authentication

### Enable 2FA

```bash
moltbot auth enable-2fa
```

Output:
```
Scan this QR code with your authenticator app:
[QR CODE]

Or enter this key manually: ABCD1234EFGH5678

Enter code to verify: _
```

### Require 2FA for Dashboard

```json
{
  "gateway": {
    "auth": {
      "require2FA": true
    }
  }
}
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| "Invalid password" | Reset with `moltbot config set gateway.auth.password` |
| API key rejected | Check key hasn't been revoked |
| OAuth redirect fails | Verify callback URL in provider |
| Pairing code expired | Generate new code with `moltbot auth pair` |

### Reset Authentication

```bash
# Reset password
moltbot auth reset-password

# Clear all auth data
moltbot auth reset --all

# Re-run setup
moltbot onboard --auth-only
```

## Security Checklist

| Setting | Recommended |
|---------|-------------|
| Dashboard password | Strong, unique |
| API keys | Rotate every 90 days |
| DM policy | Pairing or whitelist |
| 2FA | Enabled for admins |
| Session timeout | 24 hours max |
