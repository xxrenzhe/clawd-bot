## Google Calendar Setup

### Step 1: Enable Google Calendar API

1. Go to [console.cloud.google.com](https://console.cloud.google.com)
2. Create or select a project
3. Go to "APIs & Services" ‚Üí "Enable APIs"
4. Search for "Google Calendar API"
5. Click "Enable"

### Step 2: Create OAuth Credentials

1. Go to "APIs & Services" ‚Üí "Credentials"
2. Click "Create Credentials" ‚Üí "OAuth client ID"
3. Application type: "Desktop app"
4. Name: "Moltbot Calendar"
5. Download the JSON file
6. Save as `~/.moltbot/google-credentials.json`

### Step 3: Install Calendar Skill

```bash
moltbot skill add calendar
```

### Step 4: Authenticate

```bash
moltbot auth google-calendar
```

This opens a browser for OAuth consent. Grant calendar permissions.

<HostingCTA context="inline" />

### Step 5: Configure

```json
{
  "skills": {
    "calendar": {
      "provider": "google",
      "credentialsPath": "~/.moltbot/google-credentials.json",
      "defaultCalendar": "primary",
      "timezone": "America/New_York"
    }
  }
}
```

## Outlook/Microsoft 365 Setup

### Step 1: Register App

1. Go to [portal.azure.com](https://portal.azure.com)
2. Azure Active Directory ‚Üí App registrations ‚Üí New
3. Set redirect URI: `http://localhost:3000/callback`
4. Note the Application (client) ID

### Step 2: Add Permissions

1. API permissions ‚Üí Add ‚Üí Microsoft Graph
2. Add: `Calendars.Read`, `Calendars.ReadWrite`
3. Grant admin consent

### Step 3: Configure Moltbot

```json
{
  "skills": {
    "calendar": {
      "provider": "outlook",
      "clientId": "your-client-id",
      "tenantId": "your-tenant-id"
    }
  }
}
```

### Step 4: Authenticate

```bash
moltbot auth outlook-calendar
```

## Security Considerations

| Setting | Recommendation |
|---------|----------------|
| OAuth scope | Minimum required (read-only first) |
| Calendar access | Only needed calendars |
| Meeting details | Consider privacy in shared spaces |

### Read-Only Mode

Start with read-only for safety:

```json
{
  "skills": {
    "calendar": {
      "permissions": "read-only"
    }
  }
}
```

Upgrade to read-write when comfortable.

## Troubleshooting

| Issue | Solution |
|-------|----------|
| "Token expired" | Run `moltbot auth google-calendar` again |
| Wrong timezone | Set `timezone` in config |
| Missing events | Check `defaultCalendar` setting |
| Can't create events | Verify read-write permissions |

**Test connection:**
```bash
moltbot skill test calendar list-events
```
