## AWS EC2 Deployment

### Step 1: Launch Instance

1. Go to EC2 Console → Launch Instance
2. Choose:
   - **AMI:** Ubuntu 24.04 LTS
   - **Instance type:** t3.small (2 vCPU, 2 GB RAM)
   - **Key pair:** Create or select existing
   - **Security group:** SSH (22) + custom (18789)
3. Launch instance

### Step 2: Connect and Install

```bash
# Connect
ssh -i your-key.pem ubuntu@your-instance-ip

# Update system
sudo apt update && sudo apt upgrade -y

# Install Node.js
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs

# Install Moltbot
curl -fsSL https://molt.bot/install.sh | bash
moltbot onboard
```

<HostingCTA context="inline" />

### Step 3: Security Group

Configure inbound rules:

| Type | Port | Source |
|------|------|--------|
| SSH | 22 | Your IP |
| Custom TCP | 18789 | 127.0.0.1 |

**Important:** Keep 18789 localhost-only. Use Cloudflare Tunnel for remote access.

### Step 4: systemd Service

```bash
sudo nano /etc/systemd/system/moltbot.service
```

```ini
[Unit]
Description=Moltbot AI Gateway
After=network.target

[Service]
Type=simple
User=ubuntu
Environment=ANTHROPIC_API_KEY=sk-ant-...
ExecStart=/usr/bin/moltbot gateway
Restart=always

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable moltbot
sudo systemctl start moltbot
```

## Google Cloud Platform

### Step 1: Create VM

```bash
gcloud compute instances create moltbot-vm \
  --zone=us-central1-a \
  --machine-type=e2-small \
  --image-family=ubuntu-2404-lts-amd64 \
  --image-project=ubuntu-os-cloud \
  --boot-disk-size=20GB
```

### Step 2: Connect and Install

```bash
gcloud compute ssh moltbot-vm --zone=us-central1-a
```

Then follow the standard install:
```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs
curl -fsSL https://molt.bot/install.sh | bash
```

### Step 3: Firewall Rules

```bash
# Block external access to gateway
gcloud compute firewall-rules create deny-moltbot-external \
  --direction=INGRESS \
  --action=DENY \
  --rules=tcp:18789 \
  --source-ranges=0.0.0.0/0
```

## Azure VM

### Step 1: Create VM

```bash
az vm create \
  --resource-group moltbot-rg \
  --name moltbot-vm \
  --image Ubuntu2404 \
  --size Standard_B1s \
  --admin-username azureuser \
  --generate-ssh-keys
```

### Step 2: Connect and Install

```bash
ssh azureuser@$(az vm show -g moltbot-rg -n moltbot-vm -d --query publicIps -o tsv)
```

Follow standard installation steps.

### Step 3: Network Security

```bash
az network nsg rule create \
  --resource-group moltbot-rg \
  --nsg-name moltbot-vmNSG \
  --name DenyMoltbotExternal \
  --priority 100 \
  --source-address-prefixes Internet \
  --destination-port-ranges 18789 \
  --access Deny
```

## Environment Variables

### AWS Systems Manager Parameter Store

```bash
# Store secrets
aws ssm put-parameter --name "/moltbot/ANTHROPIC_API_KEY" \
  --value "sk-ant-..." --type SecureString

# Retrieve in instance
aws ssm get-parameter --name "/moltbot/ANTHROPIC_API_KEY" \
  --with-decryption --query 'Parameter.Value' --output text
```

### GCP Secret Manager

```bash
# Create secret
echo -n "sk-ant-..." | gcloud secrets create anthropic-key --data-file=-

# Access in VM
gcloud secrets versions access latest --secret="anthropic-key"
```

### Azure Key Vault

```bash
# Create vault and secret
az keyvault create --name moltbot-vault --resource-group moltbot-rg
az keyvault secret set --vault-name moltbot-vault --name anthropic-key --value "sk-ant-..."

# Retrieve
az keyvault secret show --vault-name moltbot-vault --name anthropic-key --query value -o tsv
```

## Secure Remote Access

### Cloudflare Tunnel (Recommended)

```bash
# Install cloudflared
curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb -o cloudflared.deb
sudo dpkg -i cloudflared.deb

# Authenticate and create tunnel
cloudflared tunnel login
cloudflared tunnel create moltbot
cloudflared tunnel route dns moltbot ai.yourdomain.com
```

Configure:
```yaml
# ~/.cloudflared/config.yml
tunnel: moltbot
credentials-file: ~/.cloudflared/TUNNEL_ID.json
ingress:
  - hostname: ai.yourdomain.com
    service: http://localhost:18789
  - service: http_status:404
```

Run as service:
```bash
sudo cloudflared service install
sudo systemctl start cloudflared
```

## Auto-Scaling (Optional)

### AWS Auto Scaling Group

```yaml
# launch-template.yaml
Resources:
  MoltbotLaunchTemplate:
    Type: AWS::EC2::LaunchTemplate
    Properties:
      LaunchTemplateData:
        ImageId: ami-ubuntu-24.04
        InstanceType: t3.small
        UserData:
          Fn::Base64: |
            #!/bin/bash
            apt-get update
            curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
            apt-get install -y nodejs
            curl -fsSL https://molt.bot/install.sh | bash
            systemctl start moltbot
```


## Backup Strategy

### EBS Snapshots (AWS)

```bash
# Create snapshot
aws ec2 create-snapshot --volume-id vol-xxx --description "Moltbot backup"

# Automate with AWS Backup
aws backup create-backup-plan --backup-plan file://backup-plan.json
```

### GCP Disk Snapshots

```bash
gcloud compute disks snapshot moltbot-vm \
  --zone=us-central1-a \
  --snapshot-names=moltbot-backup-$(date +%Y%m%d)
```

| Issue | Solution |
|-------|----------|

| High latency | Choose region closer to AI API |
| Out of memory | Upgrade instance size |
| Connection timeout | Verify Cloudflare Tunnel running |
