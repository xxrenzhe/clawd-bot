## Debugging

### Error Analysis

```
You: I'm getting this error:

     TypeError: Cannot read property 'map' of undefined
     at UserList.render (UserList.js:15)

     Here's my code:
     function UserList({ users }) {
       return users.map(user => <User key={user.id} {...user} />);
     }

Moltbot: The error occurs because `users` is undefined when the
         component renders.

         **Quick fix:**
         ```jsx
         function UserList({ users = [] }) {
           return users.map(user => <User key={user.id} {...user} />);
         }
         ```

         **Better fix (with loading state):**
         ```jsx
         function UserList({ users, loading }) {
           if (loading) return <Spinner />;
           if (!users?.length) return <Empty />;
           return users.map(user => <User key={user.id} {...user} />);
         }
         ```

         **Root cause:** Check where UserList is rendered.
         The parent probably fetches users async but renders
         before the data arrives.
```

### Stack Trace Analysis

```
You: What's wrong here?

     Error: ECONNREFUSED 127.0.0.1:5432
     at TCPConnectWrap.afterConnect

Moltbot: PostgreSQL connection failed.

         **Likely causes:**
         1. PostgreSQL not running
         2. Wrong port (5432 is default)
         3. Database not accepting connections

         **Check:**
         ```bash
         # Is PostgreSQL running?
         sudo systemctl status postgresql

         # Is it listening on 5432?
         sudo lsof -i :5432

         # Can you connect manually?
         psql -U postgres -h localhost
         ```

         **Fix:** If not running:
         ```bash
         sudo systemctl start postgresql
         ```
```

## Code Review

### Security Review

```text
You: Review this for security issues:

     app.post('/login', (req, res) => {
       const { username, password } = req.body;
       const query = `SELECT * FROM users WHERE username='${username}'`;
       db.query(query, (err, result) => {
         if (result[0]?.password === password) {
           res.json({ token: jwt.sign({ id: result[0].id }, 'secret') });
         }
       });
     });

Moltbot: **Critical security issues found:**

         1. **SQL Injection** (Critical)
            `${username}` allows injection
            Fix: Use parameterized queries

         2. **Plain text passwords** (Critical)
            Comparing raw passwords
            Fix: Use bcrypt for hashing

         3. **Hardcoded JWT secret** (High)
            'secret' in code
            Fix: Use environment variable

         4. **No rate limiting** (Medium)
            Enables brute force attacks

         **Secure version:**
         ```javascript
         app.post('/login', rateLimit({ max: 5 }), async (req, res) => {
           const { username, password } = req.body;
           const [user] = await db.query(
             'SELECT * FROM users WHERE username = ?',
             [username]
           );
           if (user && await bcrypt.compare(password, user.password)) {
             res.json({
               token: jwt.sign({ id: user.id }, process.env.JWT_SECRET)
             });
           } else {
             res.status(401).json({ error: 'Invalid credentials' });
           }
         });
         ```
```

### Performance Review

```text
You: This is slow, can you help?

     async function getOrders(userId) {
       const orders = await db.orders.findAll({ userId });
       for (const order of orders) {
         order.items = await db.items.findAll({ orderId: order.id });
         order.shipping = await db.shipping.findOne({ orderId: order.id });
       }
       return orders;
     }

Moltbot: **N+1 query problem** - You're making 2N+1 queries.

         For 100 orders, that's 201 database queries.

         **Optimized version:**
         ```javascript
         async function getOrders(userId) {
           const orders = await db.orders.findAll({
             where: { userId },
             include: [
               { model: db.items, as: 'items' },
               { model: db.shipping, as: 'shipping' }
             ]
           });
           return orders;
         }
         ```

         This makes 1 query with JOINs.

         **If you can't use includes:**
         ```javascript
         async function getOrders(userId) {
           const orders = await db.orders.findAll({ userId });
           const orderIds = orders.map(o => o.id);

           const [items, shipping] = await Promise.all([
             db.items.findAll({ orderId: { $in: orderIds } }),
             db.shipping.findAll({ orderId: { $in: orderIds } })
           ]);

           // Merge data...
         }
         ```
         3 queries regardless of order count.
```

## Code Generation

### Function Generation

```text
You: Write a TypeScript function to debounce API calls

Moltbot: ```typescript
         function debounce<T extends (...args: any[]) => any>(
           fn: T,
           delay: number
         ): (...args: Parameters<T>) => void {
           let timeoutId: ReturnType<typeof setTimeout> | null = null;

           return function(...args: Parameters<T>) {
             if (timeoutId) {
               clearTimeout(timeoutId);
             }
             timeoutId = setTimeout(() => {
               fn.apply(this, args);
               timeoutId = null;
             }, delay);
           };
         }

         // Usage:
         const debouncedSearch = debounce((query: string) => {
           fetch(`/api/search?q=${query}`);
         }, 300);

         input.addEventListener('input', (e) => {
           debouncedSearch(e.target.value);
         });
         ```
```

### Test Generation

```text
You: Write tests for this function:

     function calculateTotal(items, discount = 0) {
       const subtotal = items.reduce((sum, item) =>
         sum + item.price * item.quantity, 0);
       return subtotal * (1 - discount / 100);
     }

Moltbot: ```javascript
         describe('calculateTotal', () => {
           it('calculates sum of items', () => {
             const items = [
               { price: 10, quantity: 2 },
               { price: 5, quantity: 3 }
             ];
             expect(calculateTotal(items)).toBe(35);
           });

           it('applies percentage discount', () => {
             const items = [{ price: 100, quantity: 1 }];
             expect(calculateTotal(items, 10)).toBe(90);
           });

           it('handles empty array', () => {
             expect(calculateTotal([])).toBe(0);
           });

           it('handles zero discount', () => {
             const items = [{ price: 50, quantity: 1 }];
             expect(calculateTotal(items, 0)).toBe(50);
           });

           it('handles 100% discount', () => {
             const items = [{ price: 50, quantity: 1 }];
             expect(calculateTotal(items, 100)).toBe(0);
           });
         });
         ```
```

## Code Explanation

```text
You: What does this regex do?
     /^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[@$!%*?&])[A-Za-z\d@$!%*?&]{8,}$/

Moltbot: This validates a **strong password**:

         Breaking it down:
         - `^` - Start of string
         - `(?=.*[a-z])` - At least one lowercase letter
         - `(?=.*[A-Z])` - At least one uppercase letter
         - `(?=.*\d)` - At least one digit
         - `(?=.*[@$!%*?&])` - At least one special char
         - `[A-Za-z\d@$!%*?&]{8,}` - 8+ chars from allowed set
         - `$` - End of string

         **Valid:** `Password1!`
         **Invalid:** `password` (no upper, digit, special)

         **Tip:** Consider these edge cases:
         - Unicode characters not allowed
         - Only specific special chars accepted
         - No max length limit
```

         ''.join(reversed(s))
