## Content Workflows

### Automated Blog Pipeline

```json
{
  "workflows": {
    "blog_post": {
      "trigger": "manual",
      "steps": [
        {
          "action": "research",
          "prompt": "Research {{topic}} - find statistics, trends, competitors"
        },
        {
          "action": "outline",
          "input": "{{step1.output}}",
          "prompt": "Create SEO-optimized outline targeting {{keyword}}"
        },
        {
          "action": "draft",
          "input": "{{step2.output}}",
          "prompt": "Write full article, 1500-2000 words"
        },
        {
          "action": "review",
          "input": "{{step3.output}}",
          "prompt": "Check for factual accuracy, add sources"
        }
      ]
    }
  }
}
```

## Content Configuration

```json
{
  "skills": {
    "content": {
      "defaultTone": "professional",
      "defaultLength": "medium",
      "includeSources": true,
      "seoOptimization": true
    }
  }
}
```

### Tone Options

| Tone | Use Case |
|------|----------|
| `professional` | B2B content, LinkedIn |
| `casual` | Social media, blogs |
| `friendly` | Email newsletters |
| `technical` | Documentation, tutorials |
| `persuasive` | Sales copy, landing pages |

## Best Practices

1. **Be specific about audience** - "Write for CTOs" vs "Write for everyone"

2. **Specify format** - "Use bullet points" or "Include a table"

3. **Set length expectations** - "500 words" or "Twitter-length"

4. **Request iterations** - "Make it more casual" or "Shorten by 50%"

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Content too generic | Add more context about brand voice |
| Missing facts | Enable web search skill for research |
| Wrong tone | Specify tone in config or prompt |
| Too long/short | Specify word count explicitly |
