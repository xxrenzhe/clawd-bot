# Build Custom Moltbot Skills: 5 Examples

![Creating Custom Skills for Moltbot](/images/articles/clawdbot-custom-skills.jpg)

## What Are Skills?

Skills extend Moltbot (formerly Clawdbot) with new capabilities:

| Skill Type | Example | Complexity |
|------------|---------|------------|
| API wrapper | Weather, stocks, news | Simple |
| Service integration | GitHub, Jira, Notion | Medium |
| Automation | File processing, backups | Medium |
| Complex workflow | Multi-step research | Advanced |

## Skill Structure

Every skill lives in `~/.moltbot/skills/` with this structure:

```
my-skill/
├── SKILL.md          # Required: Skill definition
├── config.json       # Optional: Default configuration
├── tools/            # Optional: Tool implementations
│   └── my-tool.js
└── README.md         # Optional: Documentation
```

## Step 1: Create Skill Directory

```bash
mkdir -p ~/.moltbot/skills/weather
cd ~/.moltbot/skills/weather
```

## Step 2: Write SKILL.md

This file defines your skill for the AI:

```markdown
# Weather Skill

Get current weather and forecasts for any location.

## Tools

### get_weather
Get current weather for a location.

**Parameters:**
- location (required): City name or coordinates
- units: "metric" or "imperial" (default: metric)

**Returns:** Current temperature, conditions, humidity

### get_forecast
Get 5-day forecast for a location.

**Parameters:**
- location (required): City name or coordinates
- days: Number of days (1-5, default: 5)

**Returns:** Daily forecasts with high/low temps

## Examples
- "What's the weather in Tokyo?"
- "Will it rain in London tomorrow?"
- "5-day forecast for New York"
```

## Step 3: Implement Tools

Create `tools/weather.js`:

```javascript
const API_KEY = process.env.WEATHER_API_KEY;
const BASE_URL = 'https://api.openweathermap.org/data/2.5';

export async function get_weather({ location, units = 'metric' }) {
  const response = await fetch(
    `${BASE_URL}/weather?q=${encodeURIComponent(location)}&units=${units}&appid=${API_KEY}`
  );

  if (!response.ok) {
    throw new Error(`Weather API error: ${response.statusText}`);
  }

  const data = await response.json();

  return {
    location: data.name,
    temperature: data.main.temp,
    feels_like: data.main.feels_like,
    conditions: data.weather[0].description,
    humidity: data.main.humidity,
    units: units === 'metric' ? '°C' : '°F'
  };
}

export async function get_forecast({ location, days = 5 }) {
  const response = await fetch(
    `${BASE_URL}/forecast?q=${encodeURIComponent(location)}&cnt=${days * 8}&appid=${API_KEY}`
  );

  const data = await response.json();

  // Process and return forecast data
  return data.list.filter((_, i) => i % 8 === 0).map(day => ({
    date: new Date(day.dt * 1000).toLocaleDateString(),
    high: day.main.temp_max,
    low: day.main.temp_min,
    conditions: day.weather[0].description
  }));
}
```

## Step 4: Add Configuration

Create `config.json`:

```json
{
  "name": "weather",
  "version": "1.0.0",
  "description": "Get weather and forecasts",
  "author": "Your Name",
  "requires": {
    "env": ["WEATHER_API_KEY"]
  },
  "tools": [
    {
      "name": "get_weather",
      "file": "tools/weather.js",
      "function": "get_weather"
    },
    {
      "name": "get_forecast",
      "file": "tools/weather.js",
      "function": "get_forecast"
    }
  ]
}
```

For long-running tasks, use async patterns:

```javascript
export async function long_task({ query }) {
  // Report progress
  await this.progress('Starting task...');

  // Do work
  const result = await someAsyncOperation(query);

  await this.progress('Processing results...');

  // Return final result
```

## Advanced: Calling Other Skills

Skills can invoke other skills:

```javascript
export async function research({ topic }) {
  // Use web search skill
  const searchResults = await this.invoke('web-search', 'search', {
    query: topic
  });

  // Use summarization
  const summary = await this.invoke('text', 'summarize', {
    text: searchResults.content
  });

  return summary;
}
```

## Advanced: State Management

Store state between calls:

```javascript
export async function remember({ key, value }) {
  // Save to persistent storage
  await this.state.set(key, value);
  return { saved: true };
}

export async function recall({ key }) {
  // Retrieve from storage
  const value = await this.state.get(key);
  return { value };
}
```

## Security Considerations

| Practice | Why |
|----------|-----|
| Validate all inputs | Prevent injection attacks |
| Use environment variables | Don't hardcode secrets |
| Limit file system access | Principle of least privilege |
| Rate limit API calls | Prevent abuse |
| Log actions | Audit trail |

```javascript
// Input validation example
export async function process_file({ path }) {
  // Validate path is within allowed directory
  const allowedDir = process.env.MOLTBOT_FILES_DIR || '~/.moltbot/files';
  const resolvedPath = path.resolve(path);

  if (!resolvedPath.startsWith(allowedDir)) {
    throw new Error('Access denied: path outside allowed directory');
  }

  // Safe to proceed
  return await processFile(resolvedPath);
}
```

## Example Skills

### Simple: Price Checker

```javascript
export async function check_price({ ticker }) {
  const response = await fetch(`https://api.example.com/price/${ticker}`);
  const data = await response.json();
  return {
    ticker,
    price: data.price,
    change: data.change_percent
  };
}
```

### Medium: Note Taker

```javascript
export async function add_note({ content, tags = [] }) {
  const note = {
    id: Date.now(),
    content,
    tags,
    created: new Date().toISOString()
  };

  await this.state.push('notes', note);
  return { id: note.id, message: 'Note saved' };
}

export async function search_notes({ query }) {
  const notes = await this.state.get('notes') || [];
  return notes.filter(n =>
    n.content.toLowerCase().includes(query.toLowerCase())
  );
}
```
