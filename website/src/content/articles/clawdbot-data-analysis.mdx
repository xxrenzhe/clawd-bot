## SQL Queries

### Natural Language to SQL

```
You: Query: Find all customers who spent more than $10,000
     last year but haven't purchased in 90 days

Moltbot: Here's the SQL query:

         ```sql
         SELECT
           c.customer_id,
           c.name,
           c.email,
           SUM(o.total) as total_spent,
           MAX(o.order_date) as last_order
         FROM customers c
         JOIN orders o ON c.customer_id = o.customer_id
         WHERE o.order_date >= DATE_SUB(CURDATE(), INTERVAL 1 YEAR)
         GROUP BY c.customer_id, c.name, c.email
         HAVING
           SUM(o.total) > 10000
           AND MAX(o.order_date) < DATE_SUB(CURDATE(), INTERVAL 90 DAY)
         ORDER BY total_spent DESC;
         ```

         **Results (23 customers):**

         | Name | Email | Total Spent | Last Order |
         |------|-------|-------------|------------|
         | Acme Corp | john@acme.com | $45,230 | 95 days ago |
         | TechStart | lisa@techstart.io | $38,900 | 102 days ago |
         | ... | ... | ... | ... |

         üí° These are at-risk high-value customers.
         Consider a re-engagement campaign.
```

### Query Optimization

```
You: This query is slow, can you optimize it?

     SELECT * FROM orders
     WHERE customer_id IN (
       SELECT customer_id FROM customers
       WHERE country = 'US'
     )

Moltbot: **Optimization suggestions:**

         1. **Use JOIN instead of subquery:**
         ```sql
         SELECT o.*
         FROM orders o
         INNER JOIN customers c ON o.customer_id = c.customer_id
         WHERE c.country = 'US';
         ```
         Speed improvement: ~3-5x faster

         2. **Add index if missing:**
         ```sql
         CREATE INDEX idx_customers_country ON customers(country);
         CREATE INDEX idx_orders_customer ON orders(customer_id);
         ```

         3. **Select only needed columns:**
         ```sql
         SELECT o.order_id, o.total, o.order_date
         FROM orders o
         INNER JOIN customers c ON o.customer_id = c.customer_id
         WHERE c.country = 'US';
         ```

         **Before:** Full table scan, subquery execution
         **After:** Index seek, single pass

         Expected improvement: 5-10x depending on data size
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| CSV not parsing | Check encoding (UTF-8) and delimiter |
| Query timeout | Add LIMIT clause, check indexes |
| Memory error on large files | Use streaming mode or sample data |
| Chart not rendering | Ensure data is numeric, check nulls |

## Security Notes

| Setting | Recommendation |
|---------|----------------|
| Database credentials | Use environment variables |
| Query execution | Enable read-only mode for production |
| File uploads | Set size limits, validate types |
| PII in data | Enable auto-redaction for sensitive fields |
