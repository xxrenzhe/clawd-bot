# Moltbot Discord Bot: Complete Setup Guide

![Moltbot Discord Bot Setup](/images/articles/clawdbot-discord-bot.jpg)

## What You'll Build

A Discord bot powered by Claude that:
- Responds to messages in DMs and channels
- Runs automation skills on command
- Maintains conversation context
- Works with slash commands

<HostingCTA context="setup" />

## Prerequisites

| Requirement | Status |
|-------------|--------|
| Moltbot installed | Run `moltbot health` to verify |
| Discord account | With server admin access |
| Anthropic API key | Already configured in Moltbot |

If Moltbot isn't installed yet, see [Install on Mac](/articles/install-clawdbot-mac) or [Install on Linux](/articles/install-clawdbot-linux).

## Step 1: Create Discord Application

1. Go to [discord.com/developers/applications](https://discord.com/developers/applications)
2. Click "New Application"
3. Name it (e.g., "My AI Assistant")
4. Click "Create"

## Step 2: Create Bot User

1. In your application, click "Bot" in the left sidebar
2. Click "Add Bot" → "Yes, do it!"
3. Under "Token", click "Reset Token"
4. **Copy the token** — you'll need it next

**Important settings:**
- Enable "Message Content Intent" under Privileged Gateway Intents
- This allows the bot to read message content

## Step 3: Configure Bot Permissions

In the "Bot" section, scroll to "Bot Permissions" and enable:

| Permission | Purpose |
|------------|---------|
| Read Messages/View Channels | See messages |
| Send Messages | Respond to users |
| Embed Links | Rich message formatting |
| Attach Files | Send images, files |
| Read Message History | Context for conversations |
| Use Slash Commands | /ask, /skill commands |

## Step 4: Generate Invite Link

1. Click "OAuth2" → "URL Generator"
2. Under "Scopes", select:
   - `bot`
   - `applications.commands`
3. Under "Bot Permissions", select the permissions from Step 3
4. Copy the generated URL
5. Open it in your browser and add the bot to your server

## Step 5: Configure Moltbot

Add the bot token to Moltbot:

```bash
moltbot configure --section channels.discord
```

When prompted, paste your bot token.

Or edit the config directly:

```bash
nano ~/.moltbot/moltbot.json
```

Add/update the Discord section:
```json
{
  "channels": {
    "discord": {
      "enabled": true,
      "token": "YOUR_BOT_TOKEN_HERE",
      "allowedServers": ["YOUR_SERVER_ID"],
      "dmPolicy": "pairing"
    }
  }
}
```

## Step 6: Get Your Server ID

1. In Discord, enable Developer Mode:
   - User Settings → App Settings → Advanced → Developer Mode
2. Right-click your server name → "Copy Server ID"
3. Add it to `allowedServers` in the config

## Step 7: Start the Gateway

```bash
moltbot gateway
```

Or if running as a service:
```bash
sudo systemctl restart moltbot
```

## Step 8: Test the Bot

In your Discord server:
1. Send a message mentioning the bot: `@YourBot Hello!`
2. Or DM the bot directly
3. The bot should respond using Claude

## Interaction Modes

### Mention-based (default)
```
@YourBot What's the weather like today?
```

### DM conversations
Direct messages to the bot work automatically.

### Channel-specific
Configure specific channels where the bot responds to all messages:

```json
{
  "channels": {
    "discord": {
      "activeChannels": ["CHANNEL_ID_1", "CHANNEL_ID_2"]
    }
  }
}
```

## Slash Commands

Moltbot registers slash commands automatically:

| Command | Description |
|---------|-------------|
| `/ask [question]` | Ask the AI a question |
| `/skill [name]` | Run a specific skill |
| `/clear` | Clear conversation history |
| `/help` | Show available commands |

If slash commands don't appear:
1. Kick and re-invite the bot
2. Or wait up to 1 hour for Discord to sync

## Security Settings

### DM Policy Options

```json
{
  "dmPolicy": "pairing"  // Recommended
}
```

| Policy | Behavior |
|--------|----------|
| `open` | Anyone can DM the bot |
| `pairing` | Requires approval code |
| `allowlist` | Only specific user IDs |

### Pairing Mode

When someone DMs the bot:
1. Bot responds with a pairing code
2. You receive the code in your Moltbot logs
3. You approve or deny the pairing

### Limit to Specific Servers

```json
{
  "allowedServers": ["SERVER_ID_1", "SERVER_ID_2"]
}
```

## Server-Specific Personas

Create different bot personalities for different servers:

```json
{
  "channels": {
    "discord": {
      "serverConfigs": {
        "SERVER_ID": {
          "systemPrompt": "You are a helpful coding assistant for our dev team.",
          "enabledSkills": ["github", "code-review"]
        }
      }
    }
  }
}
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Bot offline | Check `moltbot gateway status` |
| Bot doesn't respond | Enable "Message Content Intent" in Discord Developer Portal |
| "Missing permissions" | Re-invite with correct permissions |
| Slash commands missing | Wait 1 hour or kick/re-invite bot |
| Token invalid | Regenerate in Discord Developer Portal |

**View Discord-specific logs:**
```bash
moltbot logs --channel discord
```

## Rate Limits

Discord has rate limits. Moltbot handles them automatically, but for busy servers:
- Consider response delays
- Use slash commands instead of mentions in high-traffic channels
- Configure `rateLimit.discord` in config if needed
