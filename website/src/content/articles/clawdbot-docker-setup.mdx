# Moltbot Docker Setup: Production-Ready Config

![Moltbot Docker Setup](/images/articles/clawdbot-docker-setup.jpg)

## Quick Start (3 Commands)

```bash
# 1. Pull the image
docker pull moltbot/moltbot:latest

# 2. Run with your API key
docker run -d --name moltbot \
  -e ANTHROPIC_API_KEY=your-key \
  -v moltbot-data:/app/data \
  moltbot/moltbot

# 3. Verify it's running
docker exec moltbot moltbot health
```

## Prerequisites

| Requirement | Notes |
|-------------|-------|
| Docker | 20.10+ (`docker --version`) |
| Docker Compose | 2.0+ (optional, recommended) |

## Production Docker Compose

Create `docker-compose.yml`:

```yaml
version: '3.8'
services:
  moltbot:
    image: moltbot/moltbot:latest
    container_name: moltbot
    restart: unless-stopped
    ports:
      - "127.0.0.1:18789:18789"
    environment:
      - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
      - TELEGRAM_BOT_TOKEN=${TELEGRAM_BOT_TOKEN}
      - GATEWAY_BIND=0.0.0.0
      - GATEWAY_AUTH_PASSWORD=${GATEWAY_PASSWORD}
    volumes:
      - ./data:/app/data
      - ./config/moltbot.json:/app/moltbot.json:ro
      - ./skills:/app/skills:ro
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"
    healthcheck:
      test: ["CMD", "moltbot", "health"]
      interval: 30s
      timeout: 10s
      retries: 3
```

## Environment File

Create `.env` (never commit this):

```bash
# API Keys
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...  # optional

# Messaging
TELEGRAM_BOT_TOKEN=123456:ABC...
DISCORD_BOT_TOKEN=...  # optional

# Security
GATEWAY_PASSWORD=your-strong-password-here
```

## Directory Structure

```
moltbot/
├── docker-compose.yml
├── .env                 # secrets (never commit)
├── config/
│   └── moltbot.json     # configuration
├── data/                # persistent data
│   ├── memory/          # conversation history
│   └── cache/           # response cache
└── skills/              # custom skills
    └── my-skill/
        └── SKILL.md
```

## Configuration File

Create `config/moltbot.json`:

```json
{
  "gateway": {
    "bind": "0.0.0.0",
    "port": 18789,
    "auth": {
      "password": "${GATEWAY_PASSWORD}"
    }
  },
  "agent": {
    "provider": "anthropic",
    "model": "claude-3-5-sonnet",
    "systemPrompt": "You are a helpful personal assistant."
  },
  "channels": {
    "telegram": {
      "enabled": true
    }
  },
  "memory": {
    "enabled": true,
    "path": "/app/data/memory"
  }
}
```

## Deployment Commands

### Start

```bash
docker-compose up -d
```

### View Logs

```bash
docker-compose logs -f moltbot
```

### Check Health

```bash
docker exec moltbot moltbot health
docker exec moltbot moltbot doctor
```

### Update to Latest

```bash
docker-compose pull
docker-compose up -d
```

### Stop

```bash
docker-compose down
```

## Multi-Service Setup

Add Redis for caching and better performance:

```yaml
version: '3.8'
services:
  moltbot:
    image: moltbot/moltbot:latest
    container_name: moltbot
    restart: unless-stopped
    depends_on:
      - redis
    environment:
      - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
      - REDIS_URL=redis://redis:6379
    volumes:
      - ./data:/app/data
    networks:
      - moltbot-net

  redis:
    image: redis:7-alpine
    container_name: moltbot-redis
    restart: unless-stopped
    volumes:
      - redis-data:/data
    networks:
      - moltbot-net
    command: redis-server --appendonly yes

networks:
  moltbot-net:
    driver: bridge

volumes:
  redis-data:
```

## Resource Limits

For production, set resource limits:

```yaml
services:
  moltbot:
    # ... other config
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 2G
        reservations:
          cpus: '0.5'
          memory: 512M
```

## Backup with Docker

### Backup Data

```bash
# Stop container first for consistency
docker-compose stop moltbot

# Create backup
tar -czf moltbot-backup-$(date +%Y%m%d).tar.gz data/ config/

# Restart
docker-compose start moltbot
```

### Automated Backup

Add to crontab:

```bash
0 3 * * * cd /path/to/moltbot && docker-compose stop && tar -czf /backups/moltbot-$(date +\%Y\%m\%d).tar.gz data/ config/ && docker-compose start
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Container won't start | Check logs: `docker-compose logs` |
| API key errors | Verify `.env` file, check quotes |
| Port already in use | Change port in docker-compose.yml |
| Permission denied | Check volume permissions |
| Out of memory | Increase memory limit |

### Common Fixes

```bash
# Reset container
docker-compose down
docker-compose up -d --force-recreate

# Clean restart (keeps data)
docker-compose down
docker volume prune -f
docker-compose up -d

# Full reset (loses data!)
docker-compose down -v
docker-compose up -d
```

## Security Checklist

| Setting | Recommendation |
|---------|----------------|
| Port binding | `127.0.0.1:18789:18789` (not `0.0.0.0`) |
| Environment file | Never commit `.env` |
| Network | Use internal Docker network |
| Updates | Regular `docker-compose pull` |
| Secrets | Use Docker secrets for production |
