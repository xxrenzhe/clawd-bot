## Single Sign-On (SSO)

### SAML Configuration

```json
{
  "enterprise": {
    "sso": {
      "provider": "saml",
      "config": {
        "entryPoint": "https://idp.company.com/saml/sso",
        "issuer": "moltbot",
        "cert": "${SAML_CERT}",
        "callbackUrl": "https://moltbot.company.com/auth/saml/callback"
      }
    }
  }
}
```

### Okta Integration

```json
{
  "enterprise": {
    "sso": {
      "provider": "okta",
      "config": {
        "domain": "company.okta.com",
        "clientId": "${OKTA_CLIENT_ID}",
        "clientSecret": "${OKTA_CLIENT_SECRET}"
      }
    }
  }
}
```

## Audit Logging

### Comprehensive Audit Trail

```json
{
  "enterprise": {
    "audit": {
      "enabled": true,
      "events": [
        "auth.login",
        "auth.logout",
        "auth.failed",
        "config.changed",
        "user.created",
        "user.deleted",
        "tool.called",
        "data.exported"
      ],
      "storage": {
        "type": "s3",
        "bucket": "moltbot-audit-logs",
        "retention": "7y"
      }
    }
  }
}
```

```json
{

    "type": "tool",
    "id": "email_send"

  }
```

## Role-Based Access Control

### Define Roles

```json
{
  "enterprise": {
    "rbac": {
      "roles": {
        "admin": {
          "permissions": ["*"]
        },
        "manager": {
          "permissions": [
            "users.read",
            "users.create",
            "config.read",
            "audit.read"
          ]
        },
        "user": {
          "permissions": [
            "chat",
            "tools.basic"
          ]
        }
      }
    }
  }
}
```

### Assign Roles

```bash
moltbot users set-role alice@company.com manager
moltbot users set-role bob@company.com admin
```

| Issue | Cause | Solution |
|-------|-------|----------|
| SSO not working | Misconfigured SAML | Check certificates, URLs |
