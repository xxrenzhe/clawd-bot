# Moltbot Error Handling: Debug & Fix Issues

![Moltbot Error Handling](/images/articles/clawdbot-error-handling.jpg)

## Common Error Types

| Error Type | Cause | Quick Fix |
|------------|-------|-----------|
| API Timeout | Slow provider | Increase timeout |
| Rate Limited | Too many requests | Add backoff |
| Auth Failed | Invalid key | Regenerate key |
| Connection Lost | Network issue | Auto-reconnect |

## API Errors

### Rate Limiting

Error:
```
Error: Rate limit exceeded (429)
Retry-After: 60
```

Solution:
```json
{
  "agent": {
    "rateLimit": {
      "requestsPerMinute": 50,
      "tokensPerMinute": 100000,
      "retryOn429": true
    }
  }
}
```

### Authentication Errors

Error:
```
Error: Invalid API key (401)
```

Fix:
```bash
# Check key
moltbot doctor --verbose

# Reset key
moltbot config set agent.apiKey "new-key"
```

### Timeout Errors

Error:
```
Error: Request timeout after 60000ms
```

Solutions:
```json
{
  "agent": {
    "timeout": {
      "request": 120000
    },
    "model": "claude-3-haiku"  // Faster model
  }
}
```

## Provider Fallbacks

### Configure Fallback Chain

```json
{
  "agent": {
    "providers": {
      "primary": {
        "provider": "anthropic",
        "model": "claude-3-5-sonnet"
      },
      "fallback": {
        "provider": "openai",
        "model": "gpt-4-turbo"
      },
      "emergency": {
        "provider": "ollama",
        "model": "llama3"
      }
    },
    "fallback": {
      "enabled": true,
      "triggerOn": ["timeout", "rate_limit", "error"]
    }
  }
}
```

## Channel Errors

### Telegram Errors

| Error | Cause | Solution |
|-------|-------|----------|
| 401 Unauthorized | Bad token | Regenerate with BotFather |
| 409 Conflict | Multiple instances | Stop duplicate bots |
| 429 Too Many Requests | Rate limited | Reduce message rate |

### Discord Errors

| Error | Cause | Solution |
|-------|-------|----------|
| TOKEN_INVALID | Bad token | Regenerate in Developer Portal |
| Missing Permissions | Bot lacks access | Update bot permissions |
| DISALLOWED_INTENTS | Missing intents | Enable in Developer Portal |

### Auto-Reconnect

```json
{
  "channels": {
    "telegram": {
      "reconnect": {
        "enabled": true,
        "maxAttempts": 10,
        "delay": 5000
      }
    }
  }
}
```

## Error Recovery

### Graceful Degradation

```json
{
  "agent": {
    "degradation": {
      "onError": "simplify",
      "fallbackPrompt": "I'm having technical difficulties. Please try again.",
      "disableFeatures": ["web_search", "code_execution"]
    }
  }
}
```

### Queue Failed Requests

```json
{
  "agent": {
    "queue": {
      "enabled": true,
      "retryFailed": true,
      "maxQueueSize": 100
    }
  }
}
```

## Error Logging

### Log Configuration

```json
{
  "logging": {
    "level": "info",
    "errors": {
      "file": "~/.moltbot/logs/errors.log",
      "includeStack": true,
      "maxSize": "10MB"
    }
  }
}
```

### View Error Logs

```bash
# Recent errors
moltbot logs --level error --tail 50

# Filter by type
moltbot logs --filter "rate_limit"

# Export for analysis
moltbot logs --level error --format json > errors.json
```

## Troubleshooting Guide

| Symptom | Check | Solution |
|---------|-------|----------|
| No response | `moltbot health` | Restart gateway |
| Slow responses | Provider latency | Use faster model |
| Random disconnects | Network/firewall | Enable auto-reconnect |
| High error rate | `moltbot logs` | Check API limits |

### Quick Fixes

```bash
# Restart gateway
moltbot gateway restart

# Clear cache
moltbot cache clear

# Reset configuration
moltbot config reset --keep-keys

# Full diagnostic
moltbot doctor --fix
```
