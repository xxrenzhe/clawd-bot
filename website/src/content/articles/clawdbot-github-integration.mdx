# Moltbot + GitHub: Auto Code Review in 10 Minutes

![Moltbot GitHub Integration](/images/articles/clawdbot-github-integration.jpg)

| Task | Example Command |
|------|-----------------|
| Create PR | "Create a PR for my current branch" |
| Review code | "Review the latest PR on repo X" |
| Create issue | "Create an issue about the login bug" |
| Check CI status | "What's the status of PR #123?" |
| Summarize changes | "What changed in the last 5 commits?" |
| Search code | "Find all uses of authenticateUser" |

## Prerequisites

| Requirement | Status |
|-------------|--------|
| Moltbot installed | `moltbot health` |
| GitHub account | With repo access |
| Personal Access Token | Create at github.com |

## Step 1: Create GitHub Token

1. Go to [github.com/settings/tokens](https://github.com/settings/tokens)
2. Click "Generate new token (classic)"
3. Name: "Moltbot"
4. Expiration: Choose based on security needs
5. Select scopes:

| Scope | Purpose |
|-------|---------|
| `repo` | Full repository access |
| `workflow` | GitHub Actions (optional) |
| `read:org` | Organization repos (if needed) |

6. Click "Generate token"
7. Copy the token (starts with `ghp_`)

## Step 2: Add Token to Moltbot

```bash
export GITHUB_TOKEN="ghp_..."
```

Add to shell profile:
```bash
echo 'export GITHUB_TOKEN="ghp_..."' >> ~/.zshrc
source ~/.zshrc
```

## Step 3: Install GitHub Skill

```bash
moltbot skill add github
```

Verify:
```bash
moltbot skill list
```

<HostingCTA context="inline" />

## Step 4: Configure Default Repository

```bash
moltbot configure --section skills.github
```

Or edit config:
```json
{
  "skills": {
    "github": {
      "defaultRepo": "username/repo-name",
      "defaultBranch": "main"
    }
  }
}
```

## Webhook Notifications (Optional)

Get notified when PRs are opened, CI fails, or you're mentioned.

### Step 1: Create Webhook

1. Go to your repo → Settings → Webhooks
2. Click "Add webhook"
3. Payload URL: `https://your-moltbot-domain/webhook/github`
4. Content type: `application/json`
5. Secret: Generate a secure secret
6. Select events: "Let me select individual events"
   - Pull requests
   - Issues
   - Push
   - Check runs

### Step 2: Configure Moltbot

```json
{
  "webhooks": {
    "github": {
      "secret": "your-webhook-secret",
      "notifications": {
        "pr_opened": true,
        "pr_merged": true,
        "ci_failed": true,
        "mentioned": true
      }
    }
  }
}
```

Now you'll get Telegram/Discord messages when events happen.

## Multiple Repositories

Work with multiple repos:

```
You: Check PRs on company/frontend
Moltbot: Open PRs on company/frontend:
         #123: Add dark mode (ready to merge)
         #124: Fix header layout (2 comments)
         #125: Update dependencies (CI running)
```

Configure default repos per channel:

```json
{
  "channels": {
    "discord": {
      "serverConfigs": {
        "dev-team-server": {
          "github": {
            "defaultRepo": "company/frontend"
          }
        }
      }
    }
  }
}
```

## Security Considerations

| Setting | Recommendation |
|---------|----------------|
| Token scope | Minimum required |
| Token expiration | 90 days max |
| Repo access | Only needed repos |
| Webhook secret | Strong, unique |

**Fine-grained tokens (recommended):**
1. Settings → Developer settings → Personal access tokens → Fine-grained tokens
2. Select specific repositories only
3. Grant only needed permissions

## Troubleshooting

| Issue | Solution |
|-------|----------|
| "Not found" error | Check token has `repo` scope |
| Rate limited | Use authenticated requests (token set) |
| Webhook not triggering | Verify URL is accessible, check secret |
| "Bad credentials" | Token expired, regenerate |

**Test token:**
```bash
curl -H "Authorization: token $GITHUB_TOKEN" \
  https://api.github.com/user
```
