# Moltbot Jira Automation: Issues, Sprints, Reports

![Moltbot Jira Automation](/images/articles/clawdbot-jira-automation.jpg)

| Task | Example Command |

## Prerequisites

| Requirement | Notes |
|-------------|-------|
| Moltbot installed | `moltbot health` |
| Jira Cloud or Server | With API access |
| API token | Created at id.atlassian.com |

## Step 1: Create Jira API Token

### Jira Cloud

1. Go to [id.atlassian.com/manage-profile/security/api-tokens](https://id.atlassian.com/manage-profile/security/api-tokens)
2. Click "Create API token"
3. Label: "Moltbot"
4. Copy the token

### Jira Server/Data Center

1. Go to Profile ‚Üí Personal Access Tokens
2. Create new token with appropriate permissions
3. Copy the token

## Step 2: Install Jira Skill

```bash
moltbot skill add jira
```

## Step 3: Configure

```bash
export JIRA_EMAIL="you@company.com"
export JIRA_API_TOKEN="your-token"
export JIRA_URL="https://yourcompany.atlassian.net"
```

Add to shell profile for persistence.

<HostingCTA context="inline" />

Configure Moltbot:
```json
{
  "skills": {
    "jira": {
      "url": "${JIRA_URL}",
      "email": "${JIRA_EMAIL}",
      "apiToken": "${JIRA_API_TOKEN}",
      "defaultProject": "PROJ",
      "defaultIssueType": "Task"
    }
  }
}
```

## Webhook Notifications

Get notified of Jira updates:

```json
{
  "webhooks": {
    "jira": {
      "enabled": true,
      "events": ["issue_created", "issue_updated", "sprint_started"],
      "filter": {
        "assignee": "currentUser",
        "projects": ["PROJ"]
      }
    }
  }
}
```

Notifications:
```

"Investigate memory leak in production"
```

## Security Considerations

| Setting | Recommendation |
|---------|----------------|
| API token scope | Minimum required permissions |
| Project access | Only projects you need |
| Sensitive issues | Consider exclusion rules |

### Limit to Specific Projects

```json
{
  "skills": {
    "jira": {
      "allowedProjects": ["PROJ", "TEAM"],
      "deniedProjects": ["SECURITY", "HR"]
    }
  }
}
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| "Unauthorized" | Check email and API token |
| "Project not found" | Verify project key is correct |
| "Can't transition" | Check workflow allows the transition |
| "Field required" | Configure default values in config |

**Test connection:**
```bash
moltbot skill test jira search --jql="assignee=currentUser()"
```

**List projects:**
```bash
moltbot skill test jira list-projects
```
