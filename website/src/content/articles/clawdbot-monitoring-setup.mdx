## Log Management

### View Logs

```bash
# All logs
moltbot logs

# Follow logs (live)
moltbot logs -f

# Last 100 lines
moltbot logs --tail 100

# Filter by level
moltbot logs --level error
moltbot logs --level warn
```

Configure in `moltbot.json`:

```json
{
  "logging": {
    "level": "info",
    "file": "~/.moltbot/logs/moltbot.log",
    "maxSize": "10MB",
    "maxFiles": 5,
    "compress": true

}
```

## Uptime Monitoring

### Simple Health Check Script

```bash
#!/bin/bash
# health-check.sh

HEALTH=$(curl -s http://localhost:18789/health)
STATUS=$(echo $HEALTH | jq -r '.status')

if [ "$STATUS" != "healthy" ]; then
  echo "Moltbot unhealthy at $(date)" >> /var/log/moltbot-alerts.log

  # Send alert
  curl -X POST "https://api.telegram.org/bot$BOT_TOKEN/sendMessage" \
    -d "chat_id=$ALERT_CHAT_ID" \
    -d "text=üö® Moltbot is unhealthy: $HEALTH"

  # Attempt restart
  moltbot gateway restart
fi
```

Add to cron:
```bash
*/5 * * * * /path/to/health-check.sh
```

### UptimeRobot / Uptime Kuma


## Troubleshooting

| Issue | Command |
|-------|---------|
| Check if running | `moltbot health` |
| View recent errors | `moltbot logs --level error --tail 50` |
| Check API connectivity | `moltbot doctor` |
| View resource usage | `moltbot stats` |
| Restart gateway | `moltbot gateway restart` |

### Common Issues

```bash
# High memory usage
moltbot cache clear

# Slow responses
moltbot logs --slow  # Shows requests > 5s

# Connection issues
moltbot doctor --verbose
```

## Additional Notes

Moltbot workflows work best when you keep inputs tight and outputs measurable.
Define success criteria for moltbot monitoring: logs, metrics, alerts before you automate anything significant.

For teams, document configuration changes and run clawdbot doctor after major edits.
This keeps the gateway stable and reduces debugging time when scaling up.

Key reminders:
- Keep tokens and passwords out of version control.
- Validate the gateway locally before exposing it.
- Roll out new skills in small batches and monitor logs.
- Re-check allow-lists after any channel changes.
