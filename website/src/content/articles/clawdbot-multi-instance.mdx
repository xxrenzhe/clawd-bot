# Moltbot Multi-Instance: Run Multiple Bots

![Moltbot Multi-Instance Setup](/images/articles/clawdbot-multi-instance.jpg)

## Why Multiple Instances?

| Use Case | Example |
|----------|---------|
| Personal + Work | Separate API keys, different personas |
| Dev + Production | Test changes safely |
| Different channels | Telegram + Discord isolation |
| Team members | Each person's own bot |
| Different models | Claude for coding, GPT for writing |

## Instance Architecture

```
┌─────────────────────────────────────┐
│           Your Server               │
├─────────────────────────────────────┤
│  Instance 1 (Personal)              │
│  Port: 18789                        │
│  Config: ~/.moltbot/personal/       │
│  Telegram: @my_personal_bot         │

│  Instance 2 (Work)                  │
│  Port: 18790                        │
│  Config: ~/.moltbot/work/           │
│  Slack: #ai-assistant               │
└─────────────────────────────────────┘
```

## Method 1: Separate Config Directories

### Create Instance Directories

```bash
mkdir -p ~/.moltbot/personal
mkdir -p ~/.moltbot/work
```

### Personal Instance Config

```bash
nano ~/.moltbot/personal/moltbot.json
```

```json
{
  "gateway": {
    "port": 18789
  },
  "agent": {
    "provider": "anthropic",
    "model": "claude-3-5-sonnet",
    "systemPrompt": "You are my personal assistant. Help me with daily tasks, research, and personal projects."
  },
  "channels": {
    "telegram": {
      "enabled": true,
      "token": "${TELEGRAM_PERSONAL_TOKEN}"
    }
  }
}
```

### Work Instance Config

```bash
nano ~/.moltbot/work/moltbot.json
```

```json
{
  "gateway": {
    "port": 18790
  },
  "agent": {
    "provider": "anthropic",
    "model": "claude-3-5-sonnet",
    "systemPrompt": "You are a professional work assistant. Help with code review, documentation, and project management."
  },
  "channels": {
    "slack": {
      "enabled": true,
      "token": "${SLACK_WORK_TOKEN}"
    }
  },
  "skills": {
    "github": {
      "enabled": true
    },
    "jira": {
      "enabled": true
    }
  }
}
```

### Start Each Instance

```bash
# Personal
MOLTBOT_CONFIG=~/.moltbot/personal moltbot gateway &

# Work
MOLTBOT_CONFIG=~/.moltbot/work moltbot gateway &
```

## Method 2: systemd Service Templates

Create a template service:

```bash
sudo nano /etc/systemd/system/moltbot@.service
```

```ini
[Unit]
Description=Moltbot AI Gateway (%i)
After=network.target

[Service]
Type=simple
User=moltbot
Environment=MOLTBOT_CONFIG=/home/moltbot/.moltbot/%i
EnvironmentFile=/etc/moltbot/%i.env
ExecStart=/usr/bin/moltbot gateway
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

### Create Environment Files

```bash
sudo mkdir -p /etc/moltbot

# Personal instance
sudo nano /etc/moltbot/personal.env
```

```bash
ANTHROPIC_API_KEY=sk-ant-personal-key
TELEGRAM_BOT_TOKEN=personal-bot-token
```

```bash
# Work instance
sudo nano /etc/moltbot/work.env
```

```bash
ANTHROPIC_API_KEY=sk-ant-work-key
SLACK_BOT_TOKEN=your-slack-bot-token
```

### Enable and Start

```bash
sudo systemctl enable moltbot@personal
sudo systemctl enable moltbot@work

sudo systemctl start moltbot@personal
sudo systemctl start moltbot@work
```

### Manage Instances

```bash
# Check status
sudo systemctl status moltbot@personal
sudo systemctl status moltbot@work

# View logs
sudo journalctl -u moltbot@personal -f
sudo journalctl -u moltbot@work -f

# Restart one instance
sudo systemctl restart moltbot@work
```

## Method 3: Docker Compose

```yaml
# docker-compose.yml
version: '3.8'

services:
  moltbot-personal:
    image: moltbot/moltbot:latest
    container_name: moltbot-personal
    restart: unless-stopped
    ports:
      - "127.0.0.1:18789:18789"
    environment:
      - ANTHROPIC_API_KEY=${ANTHROPIC_PERSONAL_KEY}
      - TELEGRAM_BOT_TOKEN=${TELEGRAM_PERSONAL_TOKEN}
    volumes:
      - ./configs/personal:/app/config:ro
      - personal-data:/app/data

  moltbot-work:
    image: moltbot/moltbot:latest
    container_name: moltbot-work
    restart: unless-stopped
    ports:
      - "127.0.0.1:18790:18789"
    environment:
      - ANTHROPIC_API_KEY=${ANTHROPIC_WORK_KEY}
      - SLACK_BOT_TOKEN=${SLACK_WORK_TOKEN}
    volumes:
      - ./configs/work:/app/config:ro
      - work-data:/app/data

volumes:
  personal-data:
  work-data:
```

### Docker Environment File

```bash
# .env
ANTHROPIC_PERSONAL_KEY=sk-ant-...
TELEGRAM_PERSONAL_TOKEN=123456:ABC...

ANTHROPIC_WORK_KEY=sk-ant-...
SLACK_WORK_TOKEN=xoxb-...
```

### Start All

```bash
docker-compose up -d
```

## Port Allocation

| Instance | Port | Purpose |
|----------|------|---------|
| Personal | 18789 | Default, personal use |
| Work | 18790 | Work projects |
| Development | 18791 | Testing changes |
| Family | 18792 | Shared family bot |

## Shared Resources

### Shared API Key

Multiple instances can share an API key:

```bash
export ANTHROPIC_API_KEY=sk-ant-shared
```

But different configs:
- Different system prompts
- Different skills enabled
- Different messaging channels

### Separate Memory

Each instance has isolated conversation history:

```
~/.moltbot/personal/memory/
~/.moltbot/work/memory/
```

## Monitoring Multiple Instances

### Health Check Script

```bash
#!/bin/bash
# check-instances.sh

INSTANCES=("18789:personal" "18790:work")

for instance in "${INSTANCES[@]}"; do
    port="${instance%%:*}"
    name="${instance##*:}"

    if curl -sf http://localhost:$port/health > /dev/null; then
        echo "✅ $name (port $port): healthy"
    else
        echo "❌ $name (port $port): unhealthy"
    fi
done
```

### Combined Logs

```bash
# Watch all instance logs
tail -f ~/.moltbot/*/logs/*.log
```

## Resource Considerations

| Instances | RAM | CPU | Notes |
|-----------|-----|-----|-------|
| 1-2 | 1 GB | 1 core | Light use |
| 3-5 | 2 GB | 2 cores | Moderate |
| 5+ | 4+ GB | 4+ cores | Heavy use |

Each instance uses ~200-400 MB RAM at idle.

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Port conflict | Ensure unique ports per instance |
| Wrong config loading | Verify MOLTBOT_CONFIG path |
| Memory issues | Reduce instances or add RAM |
| Token conflicts | Each Telegram/Discord bot needs unique token |

### Debug Specific Instance

```bash
MOLTBOT_CONFIG=~/.moltbot/work moltbot doctor
```
