## Gateway Optimization

### Worker Configuration

```json
{
  "gateway": {
    "workers": 4,
    "maxConnections": 500,
    "keepAlive": true,
    "timeout": 60000
  }
}
```

### Worker Sizing

| System | Workers | Connections |
|--------|---------|-------------|
| 1 CPU core | 2 | 100 |
| 2 CPU cores | 4 | 250 |
| 4 CPU cores | 8 | 500 |
| 8+ CPU cores | 16 | 1000 |

### Connection Pooling

```json
{
  "gateway": {
    "pool": {
      "enabled": true,
      "minConnections": 5,
      "maxConnections": 50,
      "idleTimeout": 30000
    }
  }
}
```

## Response Speed

### Enable Streaming

Streaming shows tokens as they arrive, dramatically improving perceived speed:

```json
{
  "agent": {
    "streaming": {
      "enabled": true,
      "chunkSize": 20,
      "flushInterval": 50
    }
  }
}
```

Without streaming: Wait 3 seconds, see full response.
With streaming: See first word in 800ms, rest flows in.

### Reduce First Token Latency

```json
{
  "agent": {
    "warmup": {
      "enabled": true,
      "interval": 300000,
      "prompt": "ping"
    },
    "prefetch": {
      "systemPrompt": true,
      "tools": true
    }
  }
}
```

Keeps model connections warm to avoid cold start delays.

## Troubleshooting

| Issue | Cause | Solution |
|-------|-------|----------|
| High latency | Cold start | Enable warmup |
| Memory growth | Cache buildup | Set cache limits |
| Slow responses | Large context | Enable summarization |
| CPU spikes | Too many workers | Reduce worker count |
| Timeouts | Network issues | Increase timeouts |

### Quick Fixes

```bash
# Force garbage collection
moltbot gc

# Clear caches
moltbot cache clear

# Restart gateway
moltbot gateway restart

# Reset to defaults
moltbot config reset --section performance
```
