## Configure Rate Limits

```json
{
  "agent": {
    "rateLimit": {
      "requestsPerMinute": 50,
      "tokensPerMinute": 40000,
      "tokensPerDay": 1000000,
      "requestsPerUser": 20
    }
  }
}
```

## Request Throttling

### Basic Throttling

```json
{
  "agent": {
    "throttle": {
      "enabled": true,
      "strategy": "sliding-window",
      "window": 60000,
      "maxRequests": 50
    }
  }
}
```

### Per-User Limits

```json
{
  "agent": {
    "throttle": {
      "perUser": {
        "enabled": true,
        "requestsPerMinute": 10,
        "tokensPerHour": 50000
      }
    }
  }
}
```

### Per-Channel Limits

```json
{
  "channels": {
    "telegram": {
      "rateLimit": {
        "requestsPerMinute": 30
      }
    },
    "discord": {
      "rateLimit": {
        "requestsPerMinute": 20,
        "perGuild": true
      }
    }
  }
}
```

## Token Budgets

### Daily Budget

```json
{
  "agent": {
    "budget": {
      "daily": {
        "inputTokens": 500000,
        "outputTokens": 100000,
        "action": "warn"
      }
    }
  }
}
```

### Monthly Budget

```json
{
  "agent": {
    "budget": {
      "monthly": {
        "maxSpend": 50.00,
        "currency": "USD",
        "action": "pause"
      }
    }
  }
}
```

### Budget Actions

| Action | Behavior |
|--------|----------|
| `warn` | Log warning, continue |
| `notify` | Send alert, continue |
| `throttle` | Slow down requests |
| `pause` | Stop until reset |
| `fallback` | Switch to cheaper model |

## Handling 429 Errors

### Auto-Retry

```json
{
  "agent": {
    "retry": {
      "on429": true,
      "maxAttempts": 3,
      "respectRetryAfter": true,
      "backoff": {
        "initial": 1000,
        "multiplier": 2,
        "max": 60000
      }
    }
  }
}
```

### Queue Requests

```json
{
  "agent": {
    "queue": {
      "enabled": true,
      "maxSize": 100,
      "timeout": 120000,
      "priority": "fifo"
    }
  }
}
```

## Provider Fallback

### Fallback Chain

```json
{
  "agent": {
    "providers": {
      "primary": {
        "provider": "anthropic",
        "model": "claude-3-5-sonnet"
      },
      "fallback": {
        "provider": "openai",
        "model": "gpt-4-turbo",
        "triggerOn": ["rate_limit", "timeout"]
      },
      "emergency": {
        "provider": "ollama",
        "model": "llama3",
        "triggerOn": ["all_failed"]
      }
    }
  }
}
```

### Model Downgrade

```json
{
  "agent": {
    "fallback": {
      "onRateLimit": {
        "action": "downgrade",
        "from": "claude-3-5-sonnet",
        "to": "claude-3-haiku"
      }
    }
  }
}
```

## Troubleshooting

| Issue | Cause | Solution |
|-------|-------|----------|
| Frequent 429s | Too many requests | Lower rate, upgrade tier |
| Requests queued | At limit | Enable fallback provider |
| High costs | No budget limits | Set daily/monthly caps |
| Slow responses | Throttling active | Check queue size |
| Users blocked | Per-user limit hit | Adjust user limits |

### Debug Rate Limits

```bash
# Check current status
moltbot ratelimit status

# View queue
moltbot queue list

# Clear queue
moltbot queue clear
```

### Reset Limits

```bash
# Reset user limits
moltbot ratelimit reset --user user_123

# Reset all limits (use carefully)
moltbot ratelimit reset --all
```
