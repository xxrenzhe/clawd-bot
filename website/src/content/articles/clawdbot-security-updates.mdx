## v2.4.1 Security Patch

### CVE-2026-0001: Authentication Bypass

**Severity**: High
**Affected versions**: 2.4.0

**Description**: Under specific conditions, the gateway authentication could be bypassed when using certain reverse proxy configurations.

**Fix**: Upgrade to v2.4.1 or later.

```bash
moltbot update
moltbot doctor --check security
```

## v2.4.0 Security Fixes

### Session Management Improvements

| Issue | Description | Status |
|-------|-------------|--------|
| Session fixation | Sessions not regenerated after auth | Fixed |
| Token expiry | Long-lived tokens vulnerable | Fixed |
| Logout incomplete | Sessions persisted after logout | Fixed |

### Configuration

```json
{
  "gateway": {
    "sessions": {
      "regenerateOnAuth": true,
      "maxAge": 86400,
      "secureOnly": true
    }
  }
}
```

## Security Best Practices

### Essential Configuration

```json
{
  "gateway": {
    "bind": "loopback",
    "auth": {
      "password": "${GATEWAY_PASSWORD}",
      "rateLimiting": {
        "enabled": true,
        "maxAttempts": 5,
        "lockoutTime": 300
      }
    }
  }
}
```

### Audit Logging

Enable security audit logs:

```json
{
  "logging": {
    "audit": {
      "enabled": true,
      "events": ["auth", "config_change", "tool_exec"],
      "retention": "90d"
    }
  }
}
```

## Update Process

### Before Updating

1. **Backup** your configuration and data
2. **Review** the changelog for breaking changes
3. **Test** in a staging environment if possible

### Secure Update

```bash
# Create backup
moltbot backup create

# Update
moltbot update

# Verify
moltbot doctor --verbose

# Restart gateway
moltbot gateway restart
```

## Security Notifications

### Subscribe to Alerts

- **GitHub Security Advisories**: Watch the repository
- **Discord**: #security-announcements channel
- **Email**: security@molt.bot mailing list

### Responsible Disclosure

Report vulnerabilities to: security@molt.bot

We follow a 90-day disclosure policy.

## Troubleshooting

| Issue | Cause | Solution |
|-------|-------|----------|
| Auth errors after update | Session format change | Clear sessions, re-authenticate |
| Config validation fails | New required fields | Run config migrate |
| Gateway won't start | Permission changes | Check file permissions |
