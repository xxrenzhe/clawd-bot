## Configuration

### Basic Configuration

The config file is at `~/.moltbot/moltbot.json`:

```json
{
  "gateway": {
    "bind": "loopback",
    "port": 18789,
    "auth": {
      "password": "${GATEWAY_PASSWORD}"
    }
  },
  "agent": {
    "provider": "anthropic",
    "model": "claude-3-5-sonnet",
    "systemPrompt": "You are a helpful personal assistant."
  },
  "channels": {
    "telegram": {
      "enabled": true,
      "dmPolicy": "pairing"
    }
  },
  "memory": {
    "enabled": true,
    "path": "~/.moltbot/memory"
  }
}
```

### Environment Variables

```bash
# API Keys
export ANTHROPIC_API_KEY="sk-ant-..."
export TELEGRAM_BOT_TOKEN="123456:ABC..."

# Security
export GATEWAY_PASSWORD="strong-password-here"
```

Add to shell profile for persistence:

```bash
# ~/.zshrc or ~/.bashrc
export ANTHROPIC_API_KEY="sk-ant-..."
```

## Security Hardening

### Essential Settings

```json
{
  "gateway": {
    "bind": "loopback",
    "auth": {
      "password": "${GATEWAY_PASSWORD}",
      "rateLimiting": {
        "enabled": true,
        "maxRequests": 100,
        "windowMs": 60000
      }
    }
  },
  "channels": {
    "telegram": {
      "dmPolicy": "pairing",
      "allowedUsers": ["your_telegram_username"]
    }
  }
}
```

### Security Checklist

| Setting | Value | Why |
|---------|-------|-----|
| `gateway.bind` | `loopback` | Prevents external access |
| `dmPolicy` | `pairing` | Requires approval for new users |
| `auth.password` | Strong password | Protects dashboard access |
| Port 18789 | Not exposed | Use reverse proxy if needed |

### Remote Access (Secure)

**Option 1: Cloudflare Tunnel**

```bash
# Install cloudflared
curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb -o cloudflared.deb
sudo dpkg -i cloudflared.deb

# Create tunnel
cloudflared tunnel login
cloudflared tunnel create moltbot
cloudflared tunnel route dns moltbot ai.yourdomain.com
```

**Option 2: Tailscale**

```bash
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up
```

Access via Tailscale IP from any device on your network.

## Running as a Service

### systemd (Linux)

Create `/etc/systemd/system/moltbot.service`:

```ini
[Unit]
Description=Moltbot AI Gateway
After=network.target

[Service]
Type=simple
User=moltbot
Environment=ANTHROPIC_API_KEY=sk-ant-...
Environment=TELEGRAM_BOT_TOKEN=123456:ABC...
ExecStart=/usr/bin/moltbot gateway
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

Enable and start:

```bash
sudo systemctl daemon-reload
sudo systemctl enable moltbot
sudo systemctl start moltbot
sudo systemctl status moltbot
```

### launchd (macOS)

Create `~/Library/LaunchAgents/com.moltbot.gateway.plist`:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.moltbot.gateway</string>
    <key>ProgramArguments</key>
    <array>
        <string>/usr/local/bin/moltbot</string>
        <string>gateway</string>
    </array>
    <key>RunAtLoad</key>
    <true/>
    <key>KeepAlive</key>
    <true/>
    <key>EnvironmentVariables</key>
    <dict>
        <key>ANTHROPIC_API_KEY</key>
        <string>sk-ant-...</string>
    </dict>
</dict>
</plist>
```

Load:

```bash
launchctl load ~/Library/LaunchAgents/com.moltbot.gateway.plist
```

## Docker Deployment

### docker-compose.yml

```yaml
version: '3.8'
services:
  moltbot:
    image: moltbot/moltbot:latest
    container_name: moltbot
    restart: unless-stopped
    ports:
      - "127.0.0.1:18789:18789"
    environment:
      - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
      - TELEGRAM_BOT_TOKEN=${TELEGRAM_BOT_TOKEN}
      - GATEWAY_BIND=0.0.0.0
    volumes:
      - ./data:/app/data
      - ./config:/app/config:ro
    healthcheck:
      test: ["CMD", "moltbot", "health"]
      interval: 30s
      timeout: 10s
      retries: 3
```

Start:

```bash
docker-compose up -d
docker-compose logs -f
```

## Backup Strategy

### Manual Backup

```bash
# Create backup
moltbot backup create --output ~/backups/moltbot-$(date +%Y%m%d).tar.gz

# Backup contents
tar -tzf backup.tar.gz
# moltbot.json
# memory/
# skills/
```

### Automated Backup

Add to crontab:

```bash
0 3 * * * /usr/local/bin/moltbot backup create --output /backups/moltbot-$(date +\%Y\%m\%d).tar.gz
```

### Cloud Backup

```bash
# S3
aws s3 cp backup.tar.gz s3://your-bucket/moltbot-backups/

# B2
b2 upload-file your-bucket backup.tar.gz moltbot-backups/
```

## Troubleshooting

| Issue | Cause | Solution |
|-------|-------|----------|
| Gateway won't start | Port in use | Change port or kill process |
| Bot not responding | Token invalid | Regenerate bot token |
| API errors | Key invalid/expired | Check key at provider console |
| High memory | Cache buildup | Clear cache, reduce retention |
| Connection refused | Bind setting | Check `gateway.bind` value |
