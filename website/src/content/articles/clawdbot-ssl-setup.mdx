# Moltbot SSL Setup: HTTPS with Let's Encrypt

![Moltbot SSL Setup](/images/articles/clawdbot-ssl-setup.jpg)

## SSL Options

| Method | Complexity | Best For |
|--------|------------|----------|
| Cloudflare Tunnel | Easiest | Any deployment |
| Caddy | Easy | VPS with domain |
| Nginx + Certbot | Medium | Production VPS |
| Tailscale HTTPS | Easy | Private access |

**Recommended:** Cloudflare Tunnel for automatic HTTPS without configuration.

<HostingCTA context="setup" />

## Option 1: Cloudflare Tunnel (Easiest)

Cloudflare Tunnel provides automatic HTTPS with zero certificate management.

### Setup

```bash
# Install
brew install cloudflare/cloudflare/cloudflared  # macOS
# or
sudo apt install cloudflared  # Linux

# Authenticate
cloudflared tunnel login

# Create tunnel
cloudflared tunnel create moltbot

# Route domain
cloudflared tunnel route dns moltbot ai.yourdomain.com
```

### Configure

```yaml
# ~/.cloudflared/config.yml
tunnel: moltbot
credentials-file: ~/.cloudflared/TUNNEL_ID.json

ingress:
  - hostname: ai.yourdomain.com
    service: http://localhost:18789
  - service: http_status:404
```

### Run

```bash
cloudflared tunnel run moltbot
```

Access: `https://ai.yourdomain.com` — HTTPS works automatically.

## Option 2: Caddy (Automatic HTTPS)

Caddy handles Let's Encrypt certificates automatically.

### Install

```bash
sudo apt install -y caddy
```

### Configure

```bash
sudo nano /etc/caddy/Caddyfile
```

```
ai.yourdomain.com {
    reverse_proxy localhost:18789
}
```

### Start

```bash
sudo systemctl enable caddy
sudo systemctl start caddy
```

Caddy automatically:
- Obtains Let's Encrypt certificate
- Configures HTTPS
- Sets up auto-renewal

## Option 3: Nginx + Certbot

For full control over SSL configuration.

### Install

```bash
sudo apt install -y nginx certbot python3-certbot-nginx
```

### Create Nginx Config

```bash
sudo nano /etc/nginx/sites-available/moltbot
```

```nginx
server {
    listen 80;
    server_name ai.yourdomain.com;

    location / {
        proxy_pass http://127.0.0.1:18789;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 300s;
    }
}
```

### Enable Site

```bash
sudo ln -s /etc/nginx/sites-available/moltbot /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### Get Certificate

```bash
sudo certbot --nginx -d ai.yourdomain.com
```

Follow prompts:
1. Enter email for renewal notices
2. Agree to terms
3. Choose redirect option (recommended)

### Verify Auto-Renewal

```bash
sudo certbot renew --dry-run
```

## Option 4: Tailscale HTTPS

Tailscale can provide HTTPS for private access.

```bash
tailscale serve https / http://localhost:18789
```

Access: `https://your-machine.tailnet-name.ts.net`

## Let's Encrypt Certificate Details

### Certificate Location

```bash
sudo ls /etc/letsencrypt/live/ai.yourdomain.com/
```

Files:
- `fullchain.pem` — Certificate + intermediates
- `privkey.pem` — Private key
- `cert.pem` — Certificate only
- `chain.pem` — Intermediate certificates

### Manual Renewal

```bash
sudo certbot renew
```

### Auto-Renewal Timer

```bash
# Check timer
sudo systemctl status certbot.timer

# View renewal schedule
sudo certbot certificates
```

## Production Nginx SSL Config

After Certbot, your config becomes:

```nginx
server {
    listen 80;
    server_name ai.yourdomain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name ai.yourdomain.com;

    ssl_certificate /etc/letsencrypt/live/ai.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/ai.yourdomain.com/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;

    # Security headers
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header Strict-Transport-Security "max-age=31536000" always;

    location / {
        proxy_pass http://127.0.0.1:18789;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300s;
    }
}
```

## Configure Moltbot for Proxy

Tell Moltbot to trust the reverse proxy:

```json
{
  "gateway": {
    "bind": "loopback",
    "trustedProxies": ["127.0.0.1"]
  }
}
```

## Verify SSL

### Browser Check

Visit `https://ai.yourdomain.com` — should show padlock icon.

### SSL Labs Test

Test at [ssllabs.com/ssltest](https://www.ssllabs.com/ssltest/)

Target: A or A+ grade.

### Command Line

```bash
curl -I https://ai.yourdomain.com/health
```

Expected:
```
HTTP/2 200
...
strict-transport-security: max-age=31536000
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Certificate not trusted | Wait 5 min, clear browser cache |
| "Connection refused" | Check Nginx is running |
| Renewal failing | Check port 80 is accessible |
| Mixed content warnings | Ensure all resources use HTTPS |
| Certificate expired | Run `sudo certbot renew` |

### Debug Certbot

```bash
sudo certbot certificates
sudo journalctl -u certbot
```

### Debug Nginx

```bash
sudo nginx -t
sudo tail -f /var/log/nginx/error.log
```

## Security Headers Checklist

| Header | Purpose | Value |
|--------|---------|-------|
| Strict-Transport-Security | Force HTTPS | max-age=31536000 |
| X-Frame-Options | Prevent clickjacking | SAMEORIGIN |
| X-Content-Type-Options | Prevent MIME sniffing | nosniff |
| Content-Security-Policy | XSS protection | default-src 'self' |
