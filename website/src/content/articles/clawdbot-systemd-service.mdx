# Moltbot systemd Service: Linux Auto-Start Guide

![Moltbot systemd Service](/images/articles/clawdbot-systemd-service.jpg)

## Why systemd?

| Feature | systemd | PM2 | Manual |
|---------|---------|-----|--------|
| Auto-start on boot | Yes | Yes | No |
| Auto-restart on crash | Yes | Yes | No |
| Log management | journald | PM2 logs | Manual |
| Native Linux | Yes | No (Node.js) | — |
| Setup complexity | Low | Low | — |

**Best for:** Linux servers, VPS deployments, production environments.

<HostingCTA context="setup" />

## Prerequisites

| Requirement | Check |
|-------------|-------|
| Linux with systemd | `systemctl --version` |
| Moltbot installed | `moltbot --version` |
| sudo access | Required for service setup |

## Step 1: Find Moltbot Path

```bash
which moltbot
```

Output example:
```
/usr/bin/moltbot
# or
/home/user/.moltbot/bin/moltbot
```

Note this path for the service file.

<HostingCTA context="inline" />

## Step 2: Create Service File

```bash
sudo nano /etc/systemd/system/moltbot.service
```

Paste this content:

```ini
[Unit]
Description=Moltbot AI Gateway
Documentation=https://docs.molt.bot
After=network.target

[Service]
Type=simple
User=YOUR_USERNAME
Group=YOUR_USERNAME
WorkingDirectory=/home/YOUR_USERNAME

# Environment variables
Environment=NODE_ENV=production
Environment=ANTHROPIC_API_KEY=sk-ant-your-key-here

# Command to run
ExecStart=/usr/bin/moltbot gateway

# Restart policy
Restart=always
RestartSec=10

# Logging
StandardOutput=journal
StandardError=journal
SyslogIdentifier=moltbot

# Security hardening (optional)
NoNewPrivileges=true
PrivateTmp=true

[Install]
WantedBy=multi-user.target
```

Replace:
- `YOUR_USERNAME` with your Linux username
- `/usr/bin/moltbot` with your actual moltbot path
- `sk-ant-your-key-here` with your API key

## Step 3: Enable and Start

```bash
# Reload systemd to read new service
sudo systemctl daemon-reload

# Enable auto-start on boot
sudo systemctl enable moltbot

# Start the service now
sudo systemctl start moltbot

# Check status
sudo systemctl status moltbot
```

Expected output:
```
● moltbot.service - Moltbot AI Gateway
     Loaded: loaded (/etc/systemd/system/moltbot.service; enabled)
     Active: active (running) since Mon 2026-01-27 14:23:45 UTC

             └─12345 /usr/bin/moltbot gateway
```

## Service Commands

| Command | Description |
|---------|-------------|
| `sudo systemctl start moltbot` | Start service |
| `sudo systemctl stop moltbot` | Stop service |
| `sudo systemctl restart moltbot` | Restart service |
| `sudo systemctl status moltbot` | Check status |
| `sudo systemctl enable moltbot` | Enable auto-start |
| `sudo systemctl disable moltbot` | Disable auto-start |

## View Logs

### Live Logs

```bash
sudo journalctl -u moltbot -f
```

### Recent Logs

```bash
# Last 100 lines
sudo journalctl -u moltbot -n 100

# Last hour
sudo journalctl -u moltbot --since "1 hour ago"

# Today only
sudo journalctl -u moltbot --since today
```

### Error Logs Only

```bash
sudo journalctl -u moltbot -p err
```

## Environment Variables

### Option 1: In Service File

```ini
[Service]
Environment=ANTHROPIC_API_KEY=sk-ant-...
Environment=TELEGRAM_BOT_TOKEN=123456:ABC...
```

### Option 2: Environment File

Create `/etc/moltbot.env`:
```bash
sudo nano /etc/moltbot.env
```

```bash
ANTHROPIC_API_KEY=sk-ant-...
TELEGRAM_BOT_TOKEN=123456:ABC...
NODE_ENV=production
```

Secure it:
```bash
sudo chmod 600 /etc/moltbot.env
sudo chown root:root /etc/moltbot.env
```

Reference in service file:
```ini
[Service]
EnvironmentFile=/etc/moltbot.env
```

Reload and restart:
```bash
sudo systemctl daemon-reload
sudo systemctl restart moltbot
```

## Dedicated User (Recommended)

For security, run under a dedicated user:

```bash
# Create user
sudo useradd -r -s /bin/false moltbot

# Create home directory
sudo mkdir -p /home/moltbot/.moltbot
sudo chown -R moltbot:moltbot /home/moltbot

# Copy config
sudo cp ~/.moltbot/moltbot.json /home/moltbot/.moltbot/
sudo chown moltbot:moltbot /home/moltbot/.moltbot/moltbot.json
```

Update service file:
```ini
[Service]
User=moltbot
Group=moltbot
WorkingDirectory=/home/moltbot
```

## Advanced Configuration

### Memory Limits

```ini
[Service]
MemoryMax=1G
MemoryHigh=800M
```

### Restart Limits

```ini
[Service]
Restart=on-failure
RestartSec=10
StartLimitIntervalSec=60
StartLimitBurst=5
```

This allows 5 restarts within 60 seconds before giving up.

### Watchdog

Detect if service becomes unresponsive:

```ini
[Service]
WatchdogSec=30
```

### Resource Limits

```ini
[Service]
CPUQuota=200%
TasksMax=100
```

## Health Check Script

Create `/usr/local/bin/moltbot-health`:

```bash
#!/bin/bash
if ! curl -sf http://localhost:18789/health > /dev/null; then
    systemctl restart moltbot
    echo "$(date): Moltbot restarted due to health check failure" >> /var/log/moltbot-health.log
fi
```

Make executable and add to cron:

```bash
sudo chmod +x /usr/local/bin/moltbot-health
sudo crontab -e
```

Add:
```
*/5 * * * * /usr/local/bin/moltbot-health
```

## Multiple Instances

For different configurations (e.g., dev/prod):

Create `/etc/systemd/system/moltbot@.service`:

```ini
[Unit]
Description=Moltbot AI Gateway (%i)
After=network.target

[Service]
Type=simple
User=moltbot
EnvironmentFile=/etc/moltbot/%i.env
ExecStart=/usr/bin/moltbot gateway --config /etc/moltbot/%i.json
Restart=always

[Install]
WantedBy=multi-user.target
```

Start instances:
```bash
sudo systemctl enable moltbot@production
sudo systemctl enable moltbot@development
sudo systemctl start moltbot@production
```

## Troubleshooting

| Issue | Check | Solution |
|-------|-------|----------|
| Service won't start | `journalctl -u moltbot -e` | Fix error shown in logs |
| Permission denied | File ownership | `chown` files to service user |
| Command not found | Path in service | Use full path to moltbot |
| Keeps restarting | Exit code | `journalctl -u moltbot -e` for errors |
| Environment not loading | EnvironmentFile | Check file path and permissions |

### Debug Mode

```bash
# Stop service
sudo systemctl stop moltbot

# Run manually with same user
sudo -u YOUR_USERNAME /usr/bin/moltbot gateway --verbose

# Check for errors, then restart service
sudo systemctl start moltbot
```

### Check Service Syntax

```bash
sudo systemd-analyze verify /etc/systemd/system/moltbot.service
```

| Use systemd when... | Use PM2 when... |

| Native integration | Node.js ecosystem |
| journald logging | PM2 monitoring |
