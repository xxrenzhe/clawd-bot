# Moltbot Team Collaboration: Multi-User Setup

![Moltbot Team Collaboration](/images/articles/clawdbot-team-collaboration.jpg)

## Team Features

| Feature | Description |
|---------|-------------|
| Multi-user access | Multiple users, one instance |
| Role-based permissions | Control who can do what |
| Shared skills | Team-wide automations |
| Audit logging | Track all activity |

<HostingCTA context="setup" />

## User Management

### Add Team Members

```bash
moltbot users add alice@team.com --role member
moltbot users add bob@team.com --role admin
```

### User Roles

| Role | Permissions |
|------|-------------|
| Admin | Full access, manage users |
| Member | Use bot, create personal skills |
| Viewer | Read-only, no tool access |

## Configuration

### Team Settings

```json
{
  "team": {
    "enabled": true,
    "name": "Acme Corp",
    "users": {
      "approval": "admin",
      "maxUsers": 50
    }
  }
}
```

### Discord Integration

```json
{
  "channels": {
    "discord": {
      "enabled": true,
      "guildId": "123456789",
      "roles": {
        "admin": ["987654321"],
        "member": ["@everyone"]
      }
    }
  }
}
```

## Permissions

### Tool Permissions

```json
{
  "permissions": {
    "roles": {
      "admin": {
        "tools": ["*"],
        "channels": ["*"]
      },
      "member": {
        "tools": ["web_search", "calendar", "email_read"],
        "channels": ["telegram", "discord"]
      },
      "viewer": {
        "tools": [],
        "channels": ["discord"]
      }
    }
  }
}
```

### Per-User Overrides

```json
{
  "permissions": {
    "users": {
      "alice@team.com": {
        "tools": ["shell_exec"],
        "note": "DevOps access"
      }
    }
  }
}
```

## Shared Skills

### Create Team Skill

```bash
moltbot skills create standup-reminder --scope team
```

```json
{
  "skills": {
    "standup-reminder": {
      "scope": "team",
      "trigger": "cron:0 9 * * 1-5",
      "action": {
        "channel": "discord:#team-standup",
        "message": "@here Time for standup!"
      }
    }
  }
}
```

### Skill Visibility

| Scope | Visible To |
|-------|------------|
| `personal` | Creator only |
| `team` | All team members |
| `public` | Anyone with access |

## Audit Logging

### Enable Audit

```json
{
  "audit": {
    "enabled": true,
    "events": [
      "user_action",
      "tool_call",
      "config_change",
      "auth_event"
    ],
    "retention": "90d"
  }
}
```

### View Logs

```bash
moltbot audit logs --user alice --since "7d"
```

## Best Practices

### Security

| Practice | Implementation |
|----------|----------------|
| Principle of least privilege | Start with minimal permissions |
| Regular audits | Review logs weekly |
| Key rotation | Rotate API keys monthly |
| Exit process | Remove users promptly |

### Onboarding

1. Create user with minimal role
2. Provide training on allowed tools
3. Monitor initial usage
4. Adjust permissions as needed

## Troubleshooting

| Issue | Cause | Solution |
|-------|-------|----------|
| User can't access | Wrong role | Check permissions |
| Skill not visible | Scope mismatch | Set scope to team |
| Audit missing events | Not configured | Enable in config |
| Rate limited | Too many users | Increase limits |
