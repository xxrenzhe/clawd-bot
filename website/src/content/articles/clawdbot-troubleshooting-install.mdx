# Moltbot Installation Troubleshooting: Fix Every Error

![Moltbot Installation Troubleshooting](/images/articles/clawdbot-troubleshooting-install.jpg)

## Quick Diagnosis

```bash
moltbot doctor
```

This command checks everything and reports issues.

<HostingCTA context="setup" />

## Common Errors and Fixes

| Error | Cause | Fix |
|-------|-------|-----|
| "command not found" | Not in PATH | Restart terminal or add to PATH |
| "EACCES permission" | npm permissions | Fix npm prefix or use nvm |
| "port already in use" | Port 18789 busy | Change port or kill process |
| "API key invalid" | Wrong/expired key | Regenerate at provider |
| "Node.js too old" | Version < 22 | Upgrade Node.js |

## "moltbot: command not found"

### Cause
Moltbot binary not in system PATH.

### Fix 1: Restart Terminal
```bash
# Close and reopen terminal, then:
moltbot --version
```

### Fix 2: Add to PATH
```bash
# Add to ~/.bashrc or ~/.zshrc
export PATH="$HOME/.moltbot/bin:$PATH"

# Reload
source ~/.bashrc  # or ~/.zshrc
```

### Fix 3: Run Directly
```bash
~/.moltbot/bin/moltbot --version
```

### Fix 4: Reinstall
```bash
curl -fsSL https://molt.bot/install.sh | bash
```

## "EACCES: permission denied"

### Cause
npm trying to write to restricted directory.

### Fix 1: Use nvm (Recommended)
```bash
# Install nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Install Node.js
nvm install 22
nvm use 22

# Reinstall moltbot
curl -fsSL https://molt.bot/install.sh | bash
```

### Fix 2: Change npm Prefix
```bash
# Create local npm directory
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'

# Add to PATH
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

# Reinstall
npm install -g moltbot
```

### Fix 3: Fix Ownership (Not Recommended)
```bash
# Only if other fixes don't work
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

## "Port 18789 already in use"

### Cause
Another process using the default port.

### Fix 1: Find and Kill Process
```bash
# Find what's using the port
lsof -i :18789

# Kill it
kill -9 <PID>
```

### Fix 2: Use Different Port
```bash
moltbot gateway --port 18790
```

Or in config:
```json
{
  "gateway": {
    "port": 18790
  }
}
```

### Fix 3: Check for Existing Instance
```bash
# Check if moltbot is already running
moltbot health

# Stop it
moltbot gateway stop
```

## "Invalid API key"

### Cause
API key is wrong, expired, or has no credits.

### Fix: Verify and Replace Key

**Anthropic:**
```bash
# Test key
curl https://api.anthropic.com/v1/messages \
  -H "x-api-key: $ANTHROPIC_API_KEY" \
  -H "content-type: application/json" \
  -d '{"model":"claude-3-5-sonnet-20241022","max_tokens":10,"messages":[{"role":"user","content":"Hi"}]}'
```

Get new key: https://console.anthropic.com/

**OpenAI:**
```bash
curl https://api.openai.com/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"
```

Get new key: https://platform.openai.com/api-keys

**Set the key:**
```bash
export ANTHROPIC_API_KEY="sk-ant-..."
moltbot doctor
```

## "Node.js version too old"

### Cause
Moltbot requires Node.js 22+.

### Check Version
```bash
node --version
```

### Fix with nvm
```bash
# Install nvm if needed
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Install Node.js 22
nvm install 22
nvm use 22
nvm alias default 22

# Verify
node --version
# Should show v22.x.x
```

### Fix on Ubuntu/Debian
```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### Fix on macOS
```bash
brew install node@22
```

## "Gateway fails to start"

### Check the Error
```bash
moltbot gateway --verbose 2>&1 | head -50
```

### Common Causes

**Missing config:**
```bash
# Create default config
moltbot onboard
```

**Corrupted config:**
```bash
# Validate JSON
cat ~/.moltbot/moltbot.json | python3 -m json.tool

# If invalid, reset
mv ~/.moltbot/moltbot.json ~/.moltbot/moltbot.json.broken
moltbot onboard
```

**Missing dependencies:**
```bash
# Reinstall
curl -fsSL https://molt.bot/install.sh | bash
```

## "Telegram bot not responding"

### Check Token
```bash
# Test token directly
curl "https://api.telegram.org/bot<YOUR_TOKEN>/getMe"
```

Should return:
```json
{"ok":true,"result":{"id":123456,"is_bot":true,"first_name":"YourBot"}}
```

### Common Issues

**Token format wrong:**
```bash
# Correct format
TELEGRAM_BOT_TOKEN="123456789:ABCdefGHIjklMNOpqrsTUVwxyz"

# NOT like this
TELEGRAM_BOT_TOKEN="bot123456789:ABC..."  # Don't include "bot" prefix
```

**Bot not started:**
```
1. Open Telegram
2. Find @BotFather
3. Send /mybots
4. Select your bot
5. Ensure it's active
```

**Webhook conflict:**
```bash
# Clear any existing webhook
curl "https://api.telegram.org/bot<TOKEN>/deleteWebhook"
```

## "Discord connection failed"

### Check Token and Permissions

```bash
# Verify bot token
moltbot doctor --channel discord
```

### Required Bot Permissions

In Discord Developer Portal:
1. OAuth2 → URL Generator
2. Select scopes: `bot`, `applications.commands`
3. Select permissions:
   - Send Messages
   - Read Message History
   - Use Slash Commands
4. Use generated URL to invite bot

### Gateway Intents

Enable in Developer Portal → Bot:
- MESSAGE CONTENT INTENT ✓
- SERVER MEMBERS INTENT ✓

## "SSL/Certificate errors"

### Cause
Outdated CA certificates or network proxy.

### Fix 1: Update Certificates
```bash
# macOS
brew install ca-certificates

# Ubuntu
sudo apt update && sudo apt install ca-certificates
```

### Fix 2: Corporate Proxy
```bash
# Set proxy
export HTTP_PROXY=http://proxy.company.com:8080
export HTTPS_PROXY=http://proxy.company.com:8080

# Or skip certificate check (not recommended for production)
export NODE_TLS_REJECT_UNAUTHORIZED=0
```

## "Memory/Performance issues"

### Check Resources
```bash
moltbot stats
```

### Reduce Memory Usage
```json
{
  "memory": {
    "maxConversations": 50,
    "pruneAfterDays": 7
  },
  "cache": {
    "maxSize": "50MB"
  }
}
```

### Clear Cache
```bash
moltbot cache clear
```

```bash
moltbot doctor --verbose --output report.txt
```

Creates detailed report for support.

## Fresh Start

If nothing works, reset completely:

```bash
# 1. Backup any important data
cp -r ~/.moltbot ~/moltbot-backup

# 2. Remove existing installation
rm -rf ~/.moltbot

# 3. Fresh install
curl -fsSL https://molt.bot/install.sh | bash

# 4. Run onboarding
moltbot onboard
```
