# Moltbot Update Guide: Safe Upgrades, Zero Downtime

![Moltbot Update Guide](/images/articles/clawdbot-update-guide.jpg)

## Update Methods

| Method | Command | Best For |
|--------|---------|----------|
| Quick update | `moltbot update` | Minor versions |
| Fresh install | `curl -fsSL https://molt.bot/install.sh \| bash` | Major versions |
| Docker | `docker-compose pull && docker-compose up -d` | Containerized |
| npm | `npm update -g moltbot` | npm installs |

## Update with Backup (Safer)

For any update, backup first:

```bash
# 1. Create backup
moltbot backup create --output ~/moltbot-backup-$(date +%Y%m%d).tar.gz

# 2. Stop gateway
moltbot gateway stop

# 3. Update
moltbot update

# 4. Verify configuration
moltbot doctor

# 5. Restart
moltbot gateway
```

## Major Version Upgrade

For major versions (1.x â†’ 2.x), use the installer:

```bash
# 1. Full backup
moltbot backup create --full

# 2. Note current config
moltbot config export > ~/moltbot-config-backup.json

# 3. Stop and update
moltbot gateway stop
curl -fsSL https://molt.bot/install.sh | bash

# 4. Migrate configuration
moltbot migrate

# 5. Verify
moltbot doctor
moltbot health

# 6. Start
moltbot gateway
```

## Docker Update

### docker-compose

```bash
# 1. Pull latest image
docker-compose pull

# 2. Restart with new image
docker-compose up -d

# 3. Verify
docker exec moltbot moltbot --version
docker exec moltbot moltbot health
```

### Docker run

```bash
# 1. Stop container
docker stop moltbot
docker rm moltbot

# 2. Pull latest
docker pull moltbot/moltbot:latest

# 3. Start with same config
docker run -d --name moltbot \
  -v moltbot-data:/app/data \
  -e ANTHROPIC_API_KEY=$ANTHROPIC_API_KEY \
  moltbot/moltbot:latest
```

## npm Update

If installed via npm:

```bash
# Check current version
npm list -g moltbot

# Update to latest
npm update -g moltbot

# Or specific version
npm install -g moltbot@2.5.0
```

## Version Pinning

Lock to a specific version:

```bash
# Install specific version
moltbot install --version 2.4.1

# Pin version (prevents auto-update)
moltbot config set updates.autoCheck false
```

## Rollback

If update causes issues:

```bash
# Quick rollback to previous version
moltbot rollback

# Or restore from backup
moltbot backup restore ~/moltbot-backup-20260128.tar.gz

# Or install specific version
curl -fsSL https://molt.bot/install.sh | bash -s -- --version 2.4.1
```

## Update Schedule

| Update Type | Frequency | Downtime |
|-------------|-----------|----------|
| Patch (2.4.x) | As needed | ~10 seconds |
| Minor (2.x.0) | Monthly | ~30 seconds |
| Major (x.0.0) | Yearly | 2-5 minutes |

## Pre-Update Checklist

| Check | Command |
|-------|---------|
| Current version | `moltbot --version` |
| Gateway status | `moltbot health` |
| Backup created | `moltbot backup create` |
| Config exported | `moltbot config export > backup.json` |
| Release notes reviewed | `moltbot update --check` |

## Post-Update Verification

```bash
# 1. Check version
moltbot --version

# 2. Verify config
moltbot doctor

# 3. Check health
moltbot health

# 4. Test message
moltbot chat "Hello, are you working?"
```

## Troubleshooting Updates

| Issue | Solution |
|-------|----------|
| Update fails | `moltbot update --force` |
| Config incompatible | `moltbot migrate` |
| Gateway won't start | Check logs: `moltbot logs --tail 50` |
| Features missing | Verify version: `moltbot --version` |
| Permissions error | `sudo moltbot update` or fix npm permissions |
