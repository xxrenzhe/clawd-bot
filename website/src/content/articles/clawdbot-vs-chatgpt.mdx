## Clawdbot Strengths
Clawdbot emphasizes self-hosting, transparent configuration, and messaging-first workflows. The Gateway on port 18789 and the onboarding wizard make it practical for always-on assistants.

## When to Choose Each
Choose Clawdbot if you want full control, local data storage, and the ability to run on your own hardware. Consider ChatGPT if you prefer a managed service or an existing ecosystem.

## Verdict
Clawdbot is the better fit when ownership and self-hosting are primary requirements. ChatGPT may be the better fit when you want a turnkey service and do not need full local control.

## Prerequisites
- Node.js 22++ (node --version)
- Supported OS: macOS, Linux, Windows (via WSL2)
- Memory: 2GB RAM minimum (4GB+ recommended for browser automation)
- Storage: 500MB for installation
Install Node.js from [nodejs.org](https://nodejs.org) if needed, then confirm your version before continuing.

## Step 1: Install and Verify Clawdbot
Use the official installer or the npm alternative. For example, the installer script is the fastest option when you are starting fresh.
> Warning: Piping a script to bash can be risky. Review the script first and never expose the gateway port to the public internet.
```bash
curl -fsSL https://clawd.bot/install.sh | bash
clawdbot onboard
clawdbot health
```
Step 1: Run the installer and confirm Node.js 22+ is available.
Step 2: Use the onboarding wizard to configure your providers.
Step 3: Run the health check to verify the gateway.

## Step 2: Configure Secure Access
Keep API keys secret, avoid committing them to git, and store them in environment variables or secure config files. JSON stands for JavaScript Object Notation, and you can store configuration there when needed.
```bash
export ANTHROPIC_API_KEY="your-key-here"
export DISCORD_BOT_TOKEN="your-token-here"
clawdbot doctor
```
If you are using a reverse proxy, configure gateway.auth.password and gateway.trustedProxies for safety.

## Step 3: Connect Messaging Channels
Telegram is the easiest starting point, and Discord is a popular next step. Follow the verified setup steps and keep bot tokens private.
```bash
clawdbot configure --section channels.telegram
clawdbot configure --section channels.discord
```
Once channels are connected, test a simple message to ensure routing works end to end.

## Verification
Start the gateway and confirm the local dashboard is reachable on the default port.
```bash
clawdbot gateway --port 18789 --verbose
clawdbot gateway status
```
If the dashboard does not load, check the port and validate your API keys.

## Troubleshooting
- Node.js version too old: Upgrade to Node.js 22+: nvm install 22 && nvm use 22.
- Permission denied during installation: Use sudo for global install or fix npm permissions.
- Gateway fails to start: Check port 18789 availability, verify API keys are set.
- Telegram/Discord bot token invalid: Regenerate token from BotFather or Discord Developer Portal.
If issues persist, run the doctor command again and review the official docs.

## Security Notes
Clawdbot can have deep system access, so treat it like an admin tool.
- Bind gateway.bind to "loopback" to prevent external exposure
- Configure gateway.auth.password for authentication
- Use Tailscale or Cloudflare Tunnel for remote access (never expose ports directly)
Never share API keys or expose the gateway port to the public internet.

## Additional Notes
Clawdbot runs well on a modern laptop, a Mac Mini for 24/7 operation, a Raspberry Pi for low power, or a small VPS for remote access. Choose the footprint that matches your uptime, latency, and privacy needs.

## Deployment Options
- Local Machine: Run on your laptop or desktop. Pros: Easy setup, Direct access. Cons: Not 24/7 unless always on.
- Mac Mini M4: Recommended for 24/7 home operation. Pros: Always on, Low power, Great performance, Supports iMessage. Cons: Hardware cost (~$599+).
- Raspberry Pi 4/5: Low-cost always-on option. Pros: Very low cost (~$35-80), Low power, Always on. Cons: Limited performance, No true sleep state.
- VPS/Cloud: Cloud deployment on providers like Hetzner, DigitalOcean, AWS. Pros: 24/7 availability, Reliable, Remote access. Cons: Monthly cost ($5+), Requires cloud knowledge.
- Docker: Containerized deployment (recommended for self-hosting). Pros: Isolated, Portable, Easy updates. Cons: Docker knowledge required.

## Configuration Checklist
- Confirm Node.js 22++ and verify with "node --version".
- Run "clawdbot onboard" and store configuration in clawdbot.json.
- Set API keys in environment variables or a secure secret manager.
- Bind the gateway to loopback and set a strong dashboard password.
- Run "clawdbot doctor" and fix any reported issues.

## Channel Setup Highlights
Telegram essentials:
- Open Telegram and search for @BotFather
- Start a chat and run /newbot command
- Follow prompts to name your bot (must end in "bot")
- Copy the bot token provided by BotFather
Discord essentials:
- Go to Discord Developer Portal: https://discord.com/developers/applications
- Create a new application
- Navigate to "Bot" section and click "Add Bot"
- Click "Reset Token" to get your bot token
