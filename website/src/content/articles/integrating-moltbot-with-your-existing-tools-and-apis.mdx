## Understanding the Moltbot Architecture

Before diving into the "how," it is essential to understand the "what." Moltbot functions as a middle layer—a gateway. 

1.  **The Brain (LLMs):** Moltbot connects to external providers like Anthropic (Claude) and OpenAI, or local providers via Ollama.
2.  **The Interface (Messaging):** It routes these brains to platforms like Telegram, Discord, Slack, and WhatsApp.
3.  **The Tools (Skills/APIs):** It uses a plugin-based system to interact with the file system, execute code, or call external APIs.

By hosting Moltbot yourself, you ensure that your API keys, conversation history, and sensitive data never leave your controlled environment.

## Getting Started: The Foundation

To integrate anything, you first need a running instance of the Moltbot gateway. If you haven't installed it yet, the process is streamlined for simplicity.

### Verified Installation Commands

Open your terminal and run the quick-install script:

```bash
curl -fsSL https://clawd.bot/install.sh | bash
```

Once installed, initialize the system and configure your basic settings:

```bash
clawdbot onboard
```

The onboarding process will prompt you for your API keys (like your Anthropic or OpenAI keys). To verify that everything is running correctly, perform a health check:

```bash
clawdbot health
```

Finally, launch the gateway. This is the component that handles incoming and outgoing requests:

```bash
clawdbot gateway --port 18789 --verbose
```

## Section 1: Connecting Messaging Platforms

The most common integration for Moltbot is connecting it to a messaging app. This allows you to interact with your AI assistant from your phone or desktop without needing a dedicated UI.

### Integrating with Telegram
Telegram is often the preferred choice for Moltbot users due to its robust Bot API.

1.  **Create a Bot:** Message `@BotFather` on Telegram to create a new bot and receive your API Token.
2.  **Configuration:** In your `clawdbot.yaml` (or via the `onboard` command), navigate to the `messaging` section.
3.  **Add Token:** Input your Telegram Token.
4.  **Pairing Mode:** For security, Moltbot defaults to a "pairing" mode. When you first message your bot, it won't respond until you approve the device in the Moltbot console.

### Integrating with Discord and Slack
For professional environments, Slack and Discord integrations allow Moltbot to function as a collaborative assistant.

*   **Slack:** You will need to create a Slack App, enable "Socket Mode," and provide the `SLACK_BOT_TOKEN` and `SLACK_APP_TOKEN`.
*   **Discord:** Create an application in the Discord Developer Portal, enable "Message Content Intent," and provide the Bot Token.

## Section 2: LLM and Provider Integration

Moltbot is "model agnostic." While it has deep optimizations for Claude (hence the name Clawdbot), it integrates seamlessly with various providers.

### External APIs (Claude & GPT-4)
To integrate external LLMs, you simply provide the API keys during the `onboard` phase. Moltbot handles the complex task of managing context windows and persistent memory via file-based storage.

### Local Integration (Ollama)
If you prefer 100% privacy, you can integrate Moltbot with **Ollama**.
1.  Ensure Ollama is running on your machine (usually at `http://localhost:11434`).
2.  In the Moltbot configuration, set the provider to `ollama`.
3.  Specify the model (e.g., `llama3` or `mistral`).

This integration is perfect for processing sensitive documents that you don't want to send to the cloud.

## Section 3: API Integration and Custom Skills

The true power of Moltbot integration lies in **Skills**. Skills are essentially Python-based plugins that allow the AI to interact with external APIs.

### How Skill Integration Works
Moltbot uses a "Tool Use" pattern. When you ask a question that requires external data (e.g., "What is the status of my latest GitHub issue?"), Moltbot identifies the appropriate skill, executes the API call, and feeds the result back to the LLM.

### Example: Creating a Simple Webhook Skill
Imagine you want Moltbot to trigger a deployment on Vercel or Netlify via a webhook. You can create a skill in the `skills/` directory:

```python
# skills/webhook_tool.py

import requests

def trigger_webhook(webhook_url: str):
    """
    Triggers a custom POST webhook for automation.
    """
    response = requests.post(webhook_url)
    if response.status_code == 200:
        return "Webhook triggered successfully!"
    return f"Failed to trigger webhook: {response.status_code}"
```

Once defined, you register this tool in your configuration. Moltbot will now "know" it has the capability to trigger webhooks when you ask it to.

## Section 4: Advanced Integration & Security

When integrating Moltbot with external tools, security is paramount. Since Moltbot can have full computer access (if enabled), you must protect the gateway.

### Security Best Practices

1.  **Loopback Binding:** To prevent your gateway from being exposed to the public internet, ensure it is bound to the loopback address.
    ```yaml
    gateway:
      bind: "127.0.0.1"
    ```
2.  **Authentication:** Always configure a password for your gateway.
    ```yaml
    gateway:
      auth:
        password: "your-secure-password"
    ```
3.  **Trusted Proxies:** If you are using a reverse proxy like Nginx or Caddy to access Moltbot remotely, configure the `trustedProxies` setting to allow the proxy's IP address.
4.  **Sandboxing:** For group chats (Discord/Slack), ensure "sandbox" mode is enabled for tools. This prevents the AI from executing potentially dangerous commands without explicit confirmation.

## Section 5: Practical Use Cases for Integration

### 1. Personal Research Assistant
Integrate Moltbot with the **ArXiv API**. When you ask about the latest papers in AI, Moltbot can fetch the abstracts, summarize them, and save the summaries to your local markdown files.

### 2. Home Automation Gateway
Connect Moltbot to **Home Assistant** via its REST API. You can then message your Moltbot on WhatsApp to "Turn off the living room lights," and Moltbot will handle the API request to your local Home Assistant instance.

### 3. CRM Data Entry
Integrate with **HubSpot or Salesforce**. You can paste a meeting transcript into the chat, and Moltbot can use a custom skill to parse the action items and update the contact record in your CRM automatically.

## Troubleshooting Your Integrations

If an integration isn't working as expected, use the built-in diagnostic tools:

*   **Check Logs:** Use `clawdbot gateway --verbose` to see real-time traffic and API errors.
*   **Verify Connectivity:** Use `curl` to test if the gateway is responding:
    ```bash
    curl http://localhost:18789/health
    ```
*   **Dependency Check:** If a custom skill is failing, ensure all required Python libraries are installed in the Moltbot environment.
