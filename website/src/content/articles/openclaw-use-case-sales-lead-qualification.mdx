## Why Use OpenClaw for Sales Automation?

In a typical sales environment, leads come in via a web form, hit a shared Slack channel, and then… they sit there. Maybe a rep sees it; maybe they don't. When they do, they have to manually research the company size, guess the intent, and type it all into a CRM. It's a recipe for bad **pipeline hygiene**.

**OpenClaw** solves this by acting as an intelligent intermediary. Because it is a self-hosted AI gateway, it can:
1. **Maintain Persistence:** It remembers context across a conversation.
2. **Access Tools:** It can call external APIs (like your CRM).
3. **Ensure Privacy:** Your lead's contact info isn't being used to train a public model on a third-party server.

## Step 1: Install OpenClaw and Connect Slack

First things first: you need the engine running. We recommend a clean VPS or a dedicated local machine. OpenClaw is designed to be lightweight and fast.

Open your terminal and run the verified installation script:

```bash
# Download and install the core binaries
curl -fsSL https://clawd.bot/install.sh | bash

# Run the interactive onboarding process
clawdbot onboard
```

During the `onboard` process, you will be prompted to connect your messaging platform. For this tutorial, select **Slack**. You’ll need to create a Slack App in the [Slack API Dashboard](https://api.slack.com/apps), grab your `SLACK_BOT_TOKEN` and `SLACK_APP_TOKEN` (for Socket Mode), and paste them into the onboarding wizard.

Once finished, verify the system is healthy:

```bash
clawdbot health
```

If everything is green, start the gateway:

```bash
clawdbot gateway --port 18789 --verbose
```

## Step 2: Post Lead Form Payloads into Slack

Most lead forms (Typeform, Webflow, etc.) can send a JSON payload via a webhook. Your goal is to get this data into the Slack channel where OpenClaw is listening.

You can use a simple bridge script or a tool like `n8n` to forward these webhooks to Slack. The message should look something like this:

> **New Lead Received:**
> Name: Jane Doe
> Email: jane@startup-tech.io
> Company: Techflow Inc
> Message: "We are looking for an enterprise solution for 50+ seats by Q3."

Once this hits the channel, OpenClaw is ready to jump in.

## Step 3: Create a Qualification Checklist

An AI agent is only as good as its instructions. We need to tell OpenClaw exactly what a "qualified lead" looks like. We’ll use a **system prompt** or a specific **skill definition** to define our criteria. In this scenario, we want to look for:

- **Company Size:** Is it a fit for our product?
- **Use Case:** Does their problem match our solution?
- **Timeline:** Are they looking to buy now or in a year?
- **Budget:** Did they mention a budget range? (Optional)

In your OpenClaw configuration (usually `config.yaml` or via the skill manager), you will define a "Lead Qualifier" persona:

```markdown
Role: You are a Sales Operations Specialist for [Your Company].
Task: Analyze inbound lead messages. 
Criteria:
1. Identify if the company size is > 10 employees.
2. Determine if the use case is 'Enterprise Integration'.
3. Assign a Lead Score (1-10).
4. Generate a concise summary for the CRM.
```

## Step 4: Configure a Skill to Call Your CRM API

This is where the magic happens. OpenClaw uses "skills"—small snippets of code or API configurations—to interact with the world. You can create a skill that sends the AI-generated summary to HubSpot, Salesforce, or Pipedrive.

Here is a conceptual example of a skill that OpenClaw uses to sync data:

```javascript
// Example: syncToCRM skill
async function syncToCRM(leadSummary, leadScore, email) {
  const response = await fetch('https://api.hubapi.com/crm/v3/objects/contacts', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${process.env.CRM_API_KEY}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      properties: {
        email: email,
        lead_summary: leadSummary,
        hubspot_score: leadScore
      }
    })
  });
  return response.ok ? "Synced to CRM" : "Sync Failed";
}
```

By hooking this skill into the OpenClaw gateway, the agent can decide to run this function automatically after it finishes qualifying the lead.

## Step 5: Validate and Go Live

Before you let the bot loose on your actual customers, run a test lead through the system.

1. Type a fake lead into your Slack channel: `"Test Lead: Bob Smith from Acme Corp, needs 100 licenses next month."`
2. Watch OpenClaw process the request in the terminal (since you used the `--verbose` flag).
3. Check your CRM to see if the note was created.

If the summary is too wordy or the score is inaccurate, tweak your system prompt. This is the beauty of **OpenClaw**—you don't need to rewrite code to change business logic; you just need to talk to it like a "smart friend."

## Security Best Practices

When handling sales leads, you are dealing with PII (Personally Identifiable Information). Security isn't optional.

- **Bind to Loopback:** If you are running a reverse proxy (like Nginx), ensure your gateway binds to `127.0.0.1` to prevent people from hitting your port 18789 directly.
- **Pairing Mode:** Use `pairing` mode for your DM policy. This ensures that only approved devices and users can interact with your OpenClaw instance.
- **Sandbox Tools:** If you allow the bot to post in group channels, ensure the tools it uses (like the CRM skill) are sandboxed to only perform specific actions.
