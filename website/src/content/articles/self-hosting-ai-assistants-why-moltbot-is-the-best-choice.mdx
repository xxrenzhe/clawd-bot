## The Case for Self-Hosted AI

Before diving into the technical specifics of Moltbot, it is essential to understand the "Why" behind the self-hosting movement.

### 1. Data Sovereignty and Privacy
When you use a standard AI web interface, every query is sent to a central server. If you are a developer pasting code snippets or a business owner discussing strategy, that data is now in someone else's hands. Moltbot acts as your personal proxy. While it can connect to cloud models like Claude 3.5 Sonnet or GPT-4o, it manages the context, the memory, and the "skills" locally. By self-hosting the gateway, you ensure that the logic of *how* your AI thinks and *what* it remembers stays on your hardware.

### 2. Persistent Memory and Context
One of the greatest limitations of standard AI interfaces is "amnesia." Once a session ends, the AI forgets who you are. Moltbot utilizes file-based memory persistence. It builds a long-term understanding of your preferences and past interactions across different platforms. Whether you message your bot on Telegram today or Discord next week, the context remains unified.

### 3. Multi-Platform Orchestration
Most AI assistants are trapped in a single tab. Moltbot breaks these walls. It is designed to be a "Gateway," meaning it can route your AI's intelligence into the messaging apps you already use. It supports:
- **Telegram**
- **Discord**
- **Slack**
- **WhatsApp**

### 4. Local LLM Integration
For those who want 100% air-gapped privacy, Moltbot integrates seamlessly with **Ollama**. This allows you to run models like Llama 3 or Mistral entirely on your own machine, using Moltbot as the sophisticated interface and management layer that Ollama lacks on its own.

## Getting Started: Installation and Setup

Setting up Moltbot is designed to be straightforward for anyone comfortable with a terminal. The developers have provided a streamlined installation script that handles dependencies and environment configuration.

### Verified Installation Commands

To begin the installation on a Linux or macOS system, run the following command:

```bash
curl -fsSL https://clawd.bot/install.sh | bash
```

Once the installation is complete, you need to configure your environment. Moltbot includes an interactive onboarding process to help you link your API keys (Anthropic, OpenAI, etc.) and choose your primary messaging platforms.

```bash
clawdbot onboard
```

### Verification and Health Checks

After onboarding, it’s vital to ensure all systems are communicating correctly. Use the built-in health check to verify your connection to LLM providers and database status:

```bash
clawdbot health
```

### Running the Gateway

The core of the experience is the Gateway. This is the persistent process that listens for incoming messages and routes them to the AI. You can run it with specific flags to monitor performance:

```bash
clawdbot gateway --port 18789 --verbose
```

The `--verbose` flag is particularly useful during your first hour of use, as it provides real-time logs of how the AI is processing your requests and utilizing its memory.

## Advanced Configuration and Security

Because Moltbot is a self-hosted gateway, security is your responsibility. The platform provides several enterprise-grade features to ensure your assistant doesn't become a vulnerability.

### 1. Network Binding
By default, you should avoid exposing your gateway port directly to the open internet. In your configuration file, ensure the `gateway.bind` setting is set to "loopback" (127.0.0.1). This forces all traffic to go through a secure tunnel or a reverse proxy like Nginx or Caddy.

### 2. Pairing Mode and DM Policy
If you are using Moltbot on platforms like Discord or Telegram, you don't want strangers being able to trigger your AI (and use your API credits). 
- Enable **Pairing Mode** in your settings.
- This requires any new device or user ID to be manually "paired" or approved by you before the bot responds.
- Configure the `DM policy` to "restrictive" to ensure only verified IDs can initiate conversations.

### 3. Reverse Proxies and Authentication
If you need to access your Moltbot gateway remotely, use a reverse proxy with `gateway.trustedProxies` configured. Additionally, always set a strong `gateway.auth.password` to prevent unauthorized access to the gateway’s internal API.

```yaml
# Example Security Config
gateway:
  bind: "127.0.0.1"
  port: 18789
  auth:
    password: "your-highly-secure-password"
  pairing:
    enabled: true
```
